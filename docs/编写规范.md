# 硬件驱动文档编写指南与分类总表

> 本文件是所有硬件经验文档的编写规范、结构模板和分类规划。
>
> 每种硬件只建立一个 Markdown 文件，文件名统一采用：
>
> ```text
> 分类-领域-模块.md
> ```
>
> 例如：
>
> ```text
> 传感器-位姿-三轴陀螺仪.md
> ```

## 1. 文档组织原则

### 1.1 一个硬件种类对应一个文件

同一种硬件的以下内容必须放在同一个文件中：

- 功能和工作原理；
- 应用场景；
- 常见型号；
- 主要参数；
- 参考价格；
- 选型建议；
- 硬件连接；
- 初始化流程；
- 驱动 API；
- 数据格式和单位；
- 校准、滤波和补偿；
- 错误处理；
- 调试方法；
- 示例代码。

不要再为同一种硬件拆分“说明文件”和“驱动文件”。

### 1.2 索引文件只负责跳转

`index.md` 只负责：

- 展示所有文档；
- 按分类和领域组织链接；
- 提供快速跳转；
- 不复制模块的详细内容；
- 不维护型号、价格和驱动说明。

### 1.3 本文件只负责写作指南和规划

`总表-硬件驱动文档分类.md` 负责：

- 解释文件命名规范；
- 规定单个模块文档的章节结构；
- 规定内容质量要求；
- 维护硬件分类总表；
- 标记已完成和规划中的模块。

具体硬件的内容必须写入对应的模块文件。

## 2. 标准文件结构

每个硬件文档建议使用以下章节：

```markdown
# 模块名称

## 1. 模块概览
## 2. 工作原理和适用场景
## 3. 常见型号和参考价格
## 4. 关键参数和选型建议
## 5. 硬件连接和电气注意事项
## 6. 通信方式或控制信号
## 7. 初始化流程
## 8. 常见驱动接口
## 9. 数据格式、单位和时序
## 10. 配置参数
## 11. 校准、补偿和滤波
## 12. 中断、DMA、FIFO 或缓存
## 13. 错误处理和异常恢复
## 14. 低功耗
## 15. 示例代码
## 16. 调试方法
## 17. 常见问题
## 18. 开发注意事项
## 19. 参考资料
```

不适用的章节可以删除，但应避免为了套模板而添加无关内容。

## 3. 各章节编写要求

### 3.1 模块概览

说明模块测量或控制的对象，以及它属于传感器、执行器、通信接口还是其他硬件。

### 3.2 工作原理和适用场景

用简洁语言说明工作方式，并列举常见应用。不要把数据手册整段复制进文档。

### 3.3 常见型号和参考价格

建议使用表格：

| 型号 | 类型 | 接口 | 主要特点 | 模块参考价 | 裸芯片参考价 |
|---|---|---|---|---:|---:|
| 示例型号 | 模块/芯片 | I2C/SPI | 简要特点 | ¥10~30 | ¥5~20 |

价格要求：

- 使用人民币估算范围；
- 注明价格是裸芯片、模块还是开发板；
- 说明价格会随渠道、品牌、批量和时间变化；
- 不把估算价格写成保证价格；
- 正式采购必须以供应商报价和原厂资料为准。

### 3.4 关键参数和选型建议

至少列出与使用场景直接相关的参数，例如：

- 供电电压；
- IO 电平；
- 测量范围；
- 分辨率；
- 精度和噪声；
- 采样率；
- 功耗；
- 温度范围；
- 接口；
- 封装；
- 供应链状态。

### 3.5 硬件连接和电气注意事项

必须说明：

- 电源和地；
- 信号电平；
- 上拉、下拉或终端电阻；
- 片选、使能和中断引脚；
- 外部电容或保护器件；
- 不能超过的电压、电流和时序限制。

### 3.6 通信方式或控制信号

说明通信协议、引脚功能、设备地址、片选方式、波特率、时钟模式或 PWM/脉冲参数。

### 3.7 初始化流程

初始化应包含：

1. 配置底层总线；
2. 检查设备是否存在；
3. 读取设备 ID 或版本；
4. 复位设备；
5. 设置工作模式；
6. 设置关键参数；
7. 清理状态、FIFO 和中断；
8. 进行必要的自检。

### 3.8 常见驱动接口

推荐区分生命周期、数据、配置和诊断接口：

```text
begin()
reset()
sleep()
wake()
read()
read_raw()
read_status()
data_ready()
set_range()
set_rate()
set_mode()
set_filter()
calibrate()
self_test()
is_connected()
```

具体命名可以根据平台调整，但同一项目应保持一致。

### 3.9 数据格式、单位和时序

必须明确：

- 返回值含义；
- 单位；
- 有符号或无符号；
- 高低字节顺序；
- 缩放系数；
- 数据更新时间；
- 最大读取间隔；
- 超时条件。

### 3.10 校准、补偿和滤波

说明校准条件、采样数量、保存方式、温度影响、滤波延迟和异常值处理。校准结果应考虑版本、校验和以及恢复默认值的方法。

### 3.11 中断、DMA、FIFO 或缓存

说明缓存容量、溢出标志、清除方式、数据帧长度、时间戳和并发访问规则。

### 3.12 错误处理和异常恢复

至少考虑：

- 设备无应答；
- 通信超时；
- 校验失败；
- 数据未准备好；
- 缓存溢出；
- 数值越界；
- 设备掉电或复位；
- 总线被占用或卡死。

应给出重试、复位、重新初始化和上报故障的策略。

### 3.13 低功耗

说明工作、待机、休眠和唤醒状态下的功耗，明确唤醒源以及唤醒后需要重新配置的内容。

### 3.14 示例代码

示例代码应：

- 能体现完整初始化流程；
- 检查初始化和通信错误；
- 标明单位；
- 避免使用不存在的 API；
- 注明代码依赖的平台或库；
- 对硬件相关参数给出修改位置。

## 4. 文档质量检查清单

提交模块文档前检查：

- [ ] 文件名符合 `分类-领域-模块.md`；
- [ ] 同一种硬件没有拆成多个文件；
- [ ] 已列出常见型号；
- [ ] 已列出参考价格范围；
- [ ] 已说明价格的不确定性；
- [ ] 已说明供电和 IO 电平；
- [ ] 已说明接口、地址或控制时序；
- [ ] 已说明初始化和设备检测；
- [ ] 已说明单位、字节序和数据格式；
- [ ] 已说明校准或参数配置；
- [ ] 已说明通信失败和异常恢复；
- [ ] 已提供最小可用示例；
- [ ] 已注明原厂数据手册是最终依据；
- [ ] `index.md` 已增加该文件的链接。
