# 三轴陀螺仪


## 1. 模块概览

三轴陀螺仪用于测量绕 X、Y、Z 三个轴的角速度，常用于机器人、无人机、云台、手柄、穿戴设备和运动检测。

它输出的是角速度，不是角度。通过积分可以估算角度，但会产生累积漂移，因此长期姿态估计通常需要与加速度计、磁力计、GNSS 或视觉定位融合。


## 2. 工作原理与适用场景

```text
GX：绕 X 轴的角速度
GY：绕 Y 轴的角速度
GZ：绕 Z 轴的角速度
```

多数芯片采用右手坐标系，但模块的具体轴向、正负旋转方向和安装方向必须以芯片数据手册为准。

常见单位：

- `°/s`：度每秒；
- `rad/s`：弧度每秒；
- 原始寄存器计数值。

## 4. 关键参数与选型

### 4.1 关键参数

| 参数 | 关注点 |
|---|---|
| 量程 | ±125、±250、±500、±1000、±2000 °/s 等 |
| 噪声 | 短期输出稳定性 |
| 零偏稳定性 | 静止时长期漂移 |
| 输出数据率 | 是否满足控制环频率 |
| 接口 | I2C 方便，SPI 通常速度和实时性更好 |
| FIFO | 是否能降低主控读取抖动和丢样 |
| 中断 | 数据就绪、运动和唤醒事件 |
| 电源 | 供电电压、IO 电平和峰值电流 |
| 温度范围 | 环境适应性和温漂 |
| 供应链 | 是否停产、是否有可靠供货 |

### 4.2 选型建议

- **入门和普通机器人**：MPU-6050、BMI160、LSM6DS3；
- **低功耗设备**：BMI270、LSM6DSL、LSM6DSO、ICM-42605；
- **高速运动和姿态控制**：ICM-20602、ICM-42605、ICM-42688-P；
- **需要绝对航向**：选择九轴 IMU 或另接磁力计，但注意电机和铁磁材料造成的磁干扰；
- **量产产品**：优先考虑原厂状态、生命周期、封装、布局要求和长期供货，不要只按模块价格选择。


## 5. 硬件连接

- 确认芯片供电电压和 IO 电平；
- 不要把 5V 信号直接接入仅支持 1.8V/3.3V 的 IO；
- I2C 需要合适的上拉电阻；
- SPI 多设备共享时，每个设备应有独立片选；
- 传感器附近避免大电流开关节点和强振动源；
- 电源旁路电容应靠近电源引脚；
- 机械安装应牢固，避免传感器自身振动；
- 量产设计应参考原厂推荐 PCB 布局。

模块不一定都能承受 5V。板载稳压器、电平转换器和 I2C 地址配置也可能因模块版本不同而变化，使用前应检查原理图或实测。


## 6. 初始化流程

常见构造方式：

```python
imu = Gyroscope(i2c, address=0x68)
```

常见初始化和诊断接口：

```python
imu.begin()
imu.reset()
imu.is_connected()
imu.is_ready()
imu.who_am_i()
imu.self_test()
```

初始化通常应完成：

1. 配置 I2C 或 SPI；
2. 检查设备 ID；
3. 复位并退出休眠；
4. 设置量程、采样率和滤波器；
5. 清理 FIFO 和中断状态；
6. 执行必要的自检。

读取接口：

```python
gx, gy, gz = imu.read_gyro()
raw_x, raw_y, raw_z = imu.read_raw_gyro()
```

配置接口：

```python
imu.set_gyro_range(500)
imu.set_sample_rate(100)
imu.set_filter_bandwidth(42)
```

状态和电源接口：

```python
imu.data_ready()
imu.sleep()
imu.wake()
imu.read_temperature()
```

不同库的函数名可能不同，具体 API 以实际驱动实现为准。


## 7. 数据格式与单位

原始数据通常是有符号整数，换算一般为：

```text
角速度 = 原始值 / 灵敏度
```

灵敏度必须以对应芯片数据手册为准。驱动必须明确：

- 单位；
- 有符号或无符号；
- 高低字节顺序；
- 缩放系数；
- 数据更新时间；
- 超时条件。

轮询示例：

```python
if imu.data_ready():
    gx, gy, gz = imu.read_gyro()
```

使用 FIFO 时要检查计数和溢出标志，确保每一帧数据长度一致；高采样率应用可使用数据就绪中断或 DMA，并使用真实时间戳计算采样周期。


## 8. 校准与滤波

校准时：

1. 设备完全静止；
2. 采集多组数据；
3. 分别计算三个轴的平均零偏；
4. 保存校准值时增加版本和校验和；
5. 温度变化较大时重新校准或做温度补偿。

```python
bias_x, bias_y, bias_z = imu.calibrate_gyro(samples=500)
```

坐标确认方法：分别绕 X、Y、Z 轴旋转，观察变化最大的通道，并用右手定则确认正负方向。不能默认不同模块的轴方向相同。

小量程通常灵敏度较高但容易饱和；大量程适合快速旋转。低通滤波可以降低噪声，但会引入延迟。


## 9. 错误处理

至少应处理：

- 设备无应答；
- `WHO_AM_I` 不匹配；
- I2C 总线卡死；
- SPI 片选时序错误；
- 通信超时；
- 数据未准备好；
- FIFO 溢出；
- 原始值越界；
- 设备掉电或复位；
- 校准期间设备移动。

推荐流程：设置通信超时，进行有限次数重试；仍失败时执行总线恢复、传感器复位和重新初始化；连续失败则向应用层报告故障，不要无限阻塞。

购买或验货时可：

1. 读取设备 ID；
2. 检查静止零偏；
3. 分别绕三个轴旋转；
4. 检查 I2C 地址；
5. 对低价或可疑模块进行批次抽检。


## 10. 低功耗

```python
imu.sleep()
# 通过 GPIO 或中断唤醒
imu.wake()
```

休眠前要确认偏置是否需要保存、FIFO 和中断状态是否会丢失，以及唤醒后是否需要重新配置寄存器。低功耗选型不能只看正常工作电流，还要比较休眠电流、唤醒延迟和低功耗数据率。

```python
        self.address = address
        self.bias = (0.0, 0.0, 0.0)

    def begin(self):

        pass

    def read_raw_gyro(self):
        pass

    def set_gyro_range(self, dps):
        pass

    def set_sample_rate(self, hz):
        pass

    def calibrate_gyro(self, samples=500):
        pass

    def get_corrected_gyro(self):
        gx, gy, gz = self.read_gyro()
        bx, by, bz = self.bias
        return gx - bx, gy - by, gz - bz

imu = Gyroscope(i2c, address=0x68)
if not imu.begin():
    raise RuntimeError("陀螺仪初始化失败")

imu.set_gyro_range(500)
imu.set_sample_rate(100)
imu.calibrate_gyro(samples=500)

while True:
    if imu.data_ready():
        gx, gy, gz = imu.get_corrected_gyro()
        print("gx=%.2f, gy=%.2f, gz=%.2f °/s" % (gx, gy, gz))
```
