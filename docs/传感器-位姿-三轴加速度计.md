# 三轴加速度计


## 1. 模块概览

三轴加速度计用于测量 X、Y、Z 三个轴向上的加速度（含重力分量），属于 MEMS 运动传感器。可用于姿态检测、计步、跌落检测、振动监测、屏幕旋转、敲击识别、运动唤醒等场景。

加速度计测量的是比力（specific force），即加速度减去重力后的矢量分量。静止时输出对应重力在各轴上的投影；运动时叠加运动加速度分量。


## 2. 工作原理与适用场景

核心是 MEMS（微机电系统）结构：一小块质量块通过柔性悬臂梁悬浮在硅基底上，加速度使其位移，电容/压阻/压电元件检测位移并转换为电压。

典型应用：
- 手机/平板屏幕自动旋转；
- 可穿戴设备计步、睡眠监测、跌落检测；
- 无人机、机器人姿态估计（与陀螺仪融合）；
- 工业振动监测、冲击检测；
- 硬盘跌落保护。

## 4. 关键参数与选型

| 参数 | 关注点 |
|---|---|
| 量程 | ±2g / ±4g / ±8g / ±16g，越大动态范围越宽但分辨率越低 |
| 分辨率/位数 | 8/10/12/14/16/20 bit，越高越精细 |
| 输出数据率 | 1 Hz ~ 6.4 kHz |
| 噪声密度 | µg/√Hz，越低越灵敏 |
| 零偏稳定性 | mg 量级，长期稳定性 |
| 灵敏度温漂 | 温度引起的输出变化 |
| 电源 | 1.71V~3.6V 常见，部分 5V 兼容 |
| IO 电平 | 1.8V / 3.3V 居多，注意与 MCU 匹配 |
| 接口 | I2C 简单，SPI 速度快 |
| FIFO | 减少主机轮询负担 |
| 中断 | 数据就绪、阈值、自由落体、敲击、运动、唤醒 |
| 功耗 | 工作电流 + 休眠电流 |
| 封装 | LGA-14 / LGA-12 / QFN |

**选型建议**：
- 入门/普通项目：ADXL345、MPU6050 中的加速度部分；
- 低功耗可穿戴：BMI270、BMA400、LIS2DH；
- 高精度测量：ADXL355；
- 高 g 冲击检测：ADXL372、H3LIS331DL；
- 工业振动：ADXL356、ADXL357。


## 5. 硬件连接

- 供电：1.8V~3.6V，常加 100nF 旁路电容靠近 VDD 引脚；
- CS 引脚：决定 I2C/SPI 模式（ADXL345 高= SPI，低= I2C）；
- I2C 上拉：SDA/SCL 接 4.7kΩ 上拉到 VDD；
- INT1/INT2：中断输出，配置为推挽或开漏；
- 5V 兼容：模块上多数带 LDO 稳压 3.3V，IO 仍以 3.3V 输出为主；
- 安装方向：以芯片数据手册标记为准，PCB 上画好 X/Y 箭头；
- 走线：敏感模拟部分远离开关电源和电机驱动线。


## 6. 通信方式

- I2C：标准 100/400 kHz，部分支持 1 MHz / 3.4 MHz；
  - ADXL345 默认地址 0x53（SDO=高）或 0x1D（SDO=低）；
  - LIS2DH 默认 0x19（SA0=高）或 0x18（SA0=低）；
- SPI：四线制，CPOL/CPHA 模式 3 或 0（查手册）；
- 中断：电平/脉冲可选，高/低电平可选。


## 7. 初始化流程

1. 配置 I2C/SPI 总线（设置速率、引脚）；
2. 读取 `WHO_AM_I` 寄存器验证设备；
3. 软复位并等待启动完成；
4. 设置量程（`±4g`/`±8g`/`±16g`）；
5. 设置输出数据率（ODR）；
6. 配置 FIFO 模式（旁路/流/触发/触发-流）；
7. 配置中断映射（数据就绪、自由落体、运动等）；
8. 清除所有中断标志；
9. 使能测量模式。

## 9. 数据格式与单位

- 单位转换：加速度 = 原始值 / 灵敏度（如 ±2g 量程 16-bit 灵敏度 = 16384 LSB/g）；
- 单位：常用 `g`（9.80665 m/s²）或 `m/s²`；
- 数据更新：ODR 决定（如 100 Hz 即每 10 ms 更新）；
- 字节序：高位在前（MSB）；
- 时序要求：读取完整一帧 6 字节（X/Y/Z）应在 ODR 周期内完成。


## 10. 校准与滤波

1. 设备静置时采集 N 个样本（建议 N≥500）；
2. 计算每轴平均值作为零偏；
3. 保存时记录版本号、CRC32；
4. 温度变化超过 10°C 时建议重新校准；
5. 6 面法（高精度需求）：分别在 ±X、±Y、±Z 朝下时记录，构造校正矩阵和偏移；
6. 校正后值 = (raw - offset) × scale。


## 11. 中断与 DMA

- 数据就绪中断（每帧 1 次）；
- FIFO：旁路 / 流 / FIFO / 触发 / 触发-流；
- 唤醒中断（运动 → 主机休眠唤醒）；
- 中断引脚：INT1/INT2 路由，独立电平/开漏配置；
- 溢出标志：水印 / 溢出，需及时读取。


## 12. 错误处理

- 总线无应答：检查地址、上拉、电源；
- WHO_AM_I 不匹配：可能是假芯片或地址错；
- 数据溢出：检查量程、ODR 是否超过物理极限；
- 长期漂移：温度变化或机械冲击导致零偏漂移；
- FIFO 溢出：降低 ODR 或提高读取频率；
- 恢复流程：重试 → 软复位 → 重新初始化 → 上报故障。


## 13. 低功耗

- 工作模式：Normal / Low Power 1/2/3/4（不同 ODR 档位）；
- 休眠电流：1 µA 以下；
- 唤醒源：运动 / FIFO 水印 / 敲击 / 自由落体；
- 唤醒后通常需要重新配置。

## 15. 调试方法

1. 读 `WHO_AM_I` 确认型号；
2. 静止读取，确认 Z 轴 ≈ 9.8 m/s²，X/Y ≈ 0；
3. 翻转模块，确认各轴读数符合右手定则；
4. 示波器观察 INT 引脚，确认数据就绪频率 = ODR；
5. 晃动模块观察波形是否跟随；
6. 长时间监测零偏稳定性；
7. 温度变化时监测输出漂移。


## 16. 常见问题

- **静止时 Z 轴不是 9.8 m/s²**：零偏未校准，或安装角度偏差；
- **模块发烫**：电源接反或短路；
- **I2C 读不到**：地址错、上拉未接、CS 引脚未拉低（ADXL345）；
- **数据跳变严重**：机械振动未隔离、电源纹波大；
- **量程饱和**：运动超过量程，提高量程或降低运动强度。
