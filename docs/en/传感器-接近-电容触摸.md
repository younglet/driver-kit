# Capacitive Touch Sensor


## 1. Module Overview

A capacitive touch sensor detects touch by sensing the capacitance change caused by a human body or object. Types:
- **Self-capacitance touch**: single electrode; detects capacitance change between the electrode and ground (body sensing)
- **Mutual capacitance touch**: dual electrodes (Tx/Rx); used for matrix touch and multi-touch
- **Resistive touch**: see the Display section
- **Dedicated touch ICs**: built-in detection algorithm
- **MCU internal touch**: some MCUs (e.g. ESP32) have built-in touch IO

Applications:
- Touch keys (replacing mechanical keys)
- Touch sliders
- Matrix keypads
- Screen touch (phones / tablets)
- Waterproof panels (kitchen appliances)
- Smart home (glass panels)
- Touch ICs (TTP229, MPR121, FT6336)


## 2. Working Principle and Use Cases

### 2.1 Self-capacitance

The touch electrode and ground form a capacitor Cp. The human body adds C_touch → total capacitance change:

```text
ΔC = C_touch - Cp
```

The measurement circuit detects the capacitance change through RC oscillation, constant-current integration, or frequency-shift detection.

### 2.2 Mutual Capacitance

The intersection of Tx and Rx electrodes forms a coupling capacitance. Touch changes the coupling → the receiver-side signal changes.

### 2.3 MCU Internal Touch

For example, ESP32 measures the touch current through an IO pull-up resistor; the time difference reflects the capacitance change.

## 5. Hardware Connection

- Touch electrodes: copper foil on PCB or ITO glass / film
- Electrode size: 5~30 mm diameter (or rectangular)
- Touch traces: keep short, away from noise sources
- Overlay: glass or acrylic (0~10 mm thickness) allowed
- Parasitic capacitance: keep 1~5 mm spacing between electrodes to prevent interference
- Grounding: solid system ground reduces false triggers
- Power: regulated + decoupling capacitors (100nF + 10µF)
- Waterproof: use matrix scanning + waterproof algorithm (raindrop vs finger identification)


## 6. Communication Method

- Digital high/low: 1 touch channel direct output
- I2C: MPR121, FT6336, etc. multi-key chips
- SPI: supported by some chips
- Interrupt output: touch events trigger INT

- 3-electrode slider: two ends as main, center as reference; outputs position percentage
- Multi-electrode slider: 12+ electrodes form a linear / circular slider
- Sensitivity: narrower electrodes are more sensitive but more noise-sensitive


## 8. Data Format and Units

- Digital output: Boolean (touched / not touched)
- I2C output: touch bitmap (e.g. MPR121 returns a 12-bit touch status)
- Slider / wheel: percentage position (0~100%)
- Matrix keypad: row / column numbers
- Touch screen: X/Y coordinates + multi-touch ID


## 9. Calibration and Filtering

- Touch IC auto-calibration (environment temperature, humidity drift)
- Read baseline at idle after startup
- Threshold: touch threshold (press) and release threshold (release)
- Waterproof algorithm: adaptive threshold, distinguish raindrop / finger
- Slider linearization: fit raw value vs physical position curve


## 10. Interrupts and DMA

- Touch interrupt: triggers on touch state change
- Multi-touch: FT6336, GT911 support up to 5~10 points
- FIFO: some chips support touch sequence recording


## 11. Error Handling

- I2C error: check address and pull-ups
- False trigger: threshold too low or electrode damp
- No response: threshold too high or electrode not connected
- Drift: large temperature / humidity changes; auto-recalibration helps


## 12. Low Power

- Single-key touch ~ 1 µA quiescent current
- Multi-key chips: mA-level when touched, µA standby
- Intermittent scanning: 1 Hz is enough

```python
    def __init__(self, bus, address=0x5A):
        self.bus = bus
        self.address = address

    def begin(self):
        pass

    def read_touched(self):
        """Return 12-bit touch bitmap."""
        pass

    def is_touched(self, channel):
        """Check touch on one channel."""
        return (self.read_touched() & (1 << channel)) != 0


mpr = MPR121(i2c, address=0x5A)
mpr.begin()
while True:
    state = mpr.read_touched()
    for i in range(12):
        if state & (1 << i):
            print("Button %d pressed" % i)
```


## 14. Debugging Methods

1. Touch the electrode and observe the INT trigger
2. Read the touch bitmap to confirm state
3. Adjust the threshold and observe sensitivity
4. Anti-interference test (phone interference, hair dryer)
5. Waterproof test (drip water)
6. Proximity test (5~10 cm distance)


## 15. Frequently Asked Questions

- **Touch unresponsive**: electrode not connected, threshold too high, electrode damp
- **False trigger**: threshold too low, ESD interference, electrode dirty
- **Slider position inaccurate**: uneven electrode shape, wrong threshold
- **Raindrop false trigger**: waterproof algorithm not enabled
- **Overlay too thick**: > 10 mm drastically reduces sensitivity — use a dedicated IC

## 17. Initialization Flow (MPR121 Example)

1. Configure I2C (default 0x5A; ADDR to VSS gives 0x5B)
2. Soft-reset
3. Configure electrode charge / discharge current and time
4. Configure touch and release thresholds
5. Configure debounce time
6. Configure auto-recalibration
7. Start touch scanning
8. Read touch status on interrupt


## 18. Anti-interference and Waterproof

- Power-supply noise: filter capacitors + software filtering
- ESD: TVS diode + series resistor
- RFI: shielding + software filtering
- Waterproof: adaptive threshold, distinguish raindrop vs finger contact
- Proximity sensing: detect objects approaching within 1~10 cm
