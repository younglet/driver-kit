# Infrared Obstacle Sensor


## 1. Module Overview

An infrared obstacle sensor uses an IR pair (one IR LED + one photodiode or phototransistor). It is used for:
- Smart car obstacle avoidance
- Robot line tracking
- Industrial product counting
- Black-line tracking
- Object proximity detection
- Automatic door sensing

Features: very low cost ($1~3), fast response, but only suitable for short range (a few cm) and qualitative detection.


## 2. Working Principle and Use Cases

The IR LED emits infrared light; the phototransistor receives the reflected light:
- No reflector → transistor off → output HIGH
- Reflector present → reflected light enters the transistor → it conducts → output LOW (depends on the circuit)

Detection range depends on the surface color, material, and distance:
- White objects: strong reflection, longer range
- Black objects: weak reflection, shorter range
- Surface texture and tilt angle also matter

## 4. Key Parameters and Selection Guide

| Parameter | Description |
|---|---|
| Detection range | a few mm ~ tens of cm (model-dependent) |
| Output type | Analog / digital |
| Supply voltage | 3.3V / 5V |
| Operating current | 10~30 mA |
| Response time | < 5 ms |
| Detection angle | 5°~30° |
| Reflectivity effect | White strong, black weak |
| Ambient light immunity | Modulated IR improves this (38 kHz) |

**Selection suggestions**:
- Car obstacle avoidance: generic module / FC-03 / TCRT5000
- Line tracking: TCRT5000 / QTR series
- Object counting: E18-D80NK / slotted pair
- Industrial detection: M18 cylindrical photoelectric switch
- Adaptive threshold: QTR-3A, QTR-8A (analog output)


## 5. Hardware Connection

- Analog output: ADC sample directly
- Digital output: through comparator (LM393), threshold adjustable
- TCRT5000 module: VCC, GND, DO (digital), AO (analog)
- Note: ambient light (sunlight, incandescent) affects the reading; use modulated IR (38 kHz) for robustness
- IR LED current-limit resistor: typically 220Ω~1kΩ
- Detection range: white object 5 cm, black object 1 cm


## 6. Communication Method

- Analog output: voltage signal (0~VCC)
- Digital output: high / low level (threshold adjustable via comparator)
- No standard protocol


## 7. Initialization Flow

1. Configure GPIO (input mode)
2. Configure ADC (if reading analog output)
3. Start periodic sampling
4. Digital output decided directly
5. Analog output decided by threshold (program or adaptive)


## 8. Driver Interfaces

```python
class IRProximity:
    def __init__(self, pin, adc=None):
        self.pin = pin  # digital
        self.adc = adc  # analog (optional)

    def begin(self):
        pass

    def is_obstacle(self):
        """Whether an obstacle is present (digital output)."""
        return self.pin.value() == 0  # assume active-low

    def read_raw(self):
        """Return ADC raw value."""
        return self.adc.read() if self.adc else None

    def read_reflectance(self):
        """0 = white, 1 = black, based on threshold."""
        return self.read_raw() < self.threshold


ir = IRProximity(digital_pin, adc_pin)
while True:
    print("Obstacle:", ir.is_obstacle())
```


## 9. Data Format and Units

- Digital output: Boolean (high / low level)
- Analog output: voltage or ADC count
- Units: no standard distance unit (qualitative only)


## 10. Calibration and Filtering

- Digital output: tune the potentiometer on the module
- Analog output: determine the threshold experimentally (ADC values for white / black)

### 10.1 Adaptive Threshold

```python
def auto_calibrate(samples):
    """Auto-calibrate threshold based on the scene."""
    max_val = max(samples)
    min_val = min(samples)
    return (max_val + min_val) / 2
```

- Digital output: multi-sample debouncing
- Analog output: median filter for spikes
- Moving average reduces jitter


## 11. Interrupts and DMA

- Digital output can be used as an interrupt / wake-up source
- 10~100 Hz periodic sampling is enough


## 12. Error Handling

- Range exceeded: always returns a fixed value
- Direct sunlight: may cause false triggers (modulated IR helps)
- Black objects: very short range; switch to laser ToF if needed
- Module failure: LED burned out


## 13. Low Power

- Turning off the LED saves significant power
- Power on only during detection (intermittent operation)


## 14. Example Code

```python
class IRProximity:
    def __init__(self, do_pin, ao_pin=None):
        self.do = do_pin
        self.ao = ao_pin
        self.threshold = 512

    def begin(self):
        pass

    def is_obstacle(self):
        """Active-low."""
        return self.do.value() == 0

    def read_ao(self):
        """Return 0~1023."""
        return self.ao.read()

    def read_color(self):
        """Black / white decision."""
        if self.ao.read() < self.threshold:
            return "black"
        return "white"


ir = IRProximity(do_pin, ao_pin)
while True:
    print("Obstacle:", ir.is_obstacle(), "Color:", ir.read_color())
```


## 15. Debugging Methods

1. Bring a white paper close and check the digital-output trigger distance
2. Bring a black paper close and check the digital-output trigger distance
3. Inspect the analog output range
4. Test in sunlight (watch for false triggers)
5. Adjust the potentiometer and watch the threshold change


## 16. Frequently Asked Questions

- **Digital output toggles frequently**: unstable threshold; use multi-sample debouncing
- **Black object not detected**: short range; switch to laser ToF
- **False triggers in sunlight**: use 38 kHz modulation or add an optical filter
- **Multiple sensors interfering with each other**: stagger the emission timing
- **TCRT5000 detection range too short**: replace the LED with a higher-power one or use a different model
