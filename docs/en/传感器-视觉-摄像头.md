# Camera Sensor


## 1. Module Overview

A camera sensor is an image-acquisition device used for photo, video, and machine vision. Common uses:
- Phone / tablet cameras
- Surveillance cameras
- Industrial vision
- Embedded vision (ESP32-CAM, OpenMV)
- Drones
- Smart home (video doorbells)
- Robotics (visual avoidance, object recognition)
- Automotive cameras

Types:
- **CMOS Image Sensor (CIS)**: mainstream
- **CCD Image Sensor**: high-end industrial / medical
- **Area / line scan**: 2D vision vs conveyor scanning
- **Color / monochrome / multi-spectral**
- **Global shutter / rolling shutter**


## 2. Working Principle and Use Cases

The image sensor converts light into electrical signals via a photodiode array:
- Photons excite electrons → form charges
- Readout circuitry converts charges into voltage
- ADC digitizes into pixel values (RGB Bayer array)

Pixel formats:
- **RGB Bayer**: raw data, requires ISP processing
- **RGB565 / YUV422**: processed data
- **JPEG / MJPEG**: compressed streams
- **H.264 / H.265**: video compression

Resolution and frame rate:
- VGA (640×480) 30 fps
- 720p (1280×720) 30 fps
- 1080p (1920×1080) 30 fps
- 4K (3840×2160) 30 fps

| Interface | DVP parallel, MIPI CSI, USB, LVDS, HDMI |
| Shutter | Global / rolling |
| Pixel size | 1.4µm ~ 3µm (affects low-light performance) |
| Dynamic range | Normal / HDR (>100 dB) |
| SNR | dB |
| Minimum illumination | lux (smaller = better for low light) |
| Auto control | AE / AWB / AF |
| Lens mount | M12 / CS / fixed focus |
| Field of view | 60° / 90° / 120° etc. |
| Aperture F | F1.2 / F2.0 etc. |

**Selection suggestions**:
- ESP32 / MCU: DVP-interface OV2640, OV7670, HM01B0
- Raspberry Pi / Linux: MIPI CSI (OV5647, IMX219, IMX477)
- Global shutter (high speed): MT9V034, OV9281, IMX296
- Low-light night vision: IMX290 + IR light
- Embedded vision development: OpenMV, HuskyLens, Maix
- Industrial machine vision: MT9V034 + FPGA/SoC


## 5. Hardware Connection

- Power: cameras usually need 1.8V (core), 2.8V (analog), 3.3V (IO); multi-rail
- Clock: external clock (24 MHz, 27 MHz, 74.25 MHz)
- Reset: pull-up on RESET pin + software reset
- PWDN: low-power control
- SCCB (I2C-like): register configuration interface
- DVP: 8/10-bit parallel; pay attention to layout (impedance matching, equal-length data lines)
- MIPI CSI: strict differential routing (100Ω±10% impedance), length matching
- Lens mounting: manual focus vs autofocus (VCM drive)
- Heat dissipation: 4K / high-frame-rate cameras need heatsinks
- ESD protection: TVS
- Dust / waterproof: sealed module


## 6. Communication Method

- **DVP (Digital Video Port)**: 8/10/12-bit parallel, PCLK + HREF + VSYNC
- **MIPI CSI-2**: differential serial, 1/2/4 lanes, up to 2.5 Gbps per lane
- **USB**: UVC (USB Video Class), driver-free on PC
- **Ethernet**: IP camera (RTSP stream)
- **LVDS / Sub-LVDS**: industrial high-speed interface
- Configuration interface: SCCB (I2C-like)


## 7. Initialization Flow

1. Power on
2. Pull RESET low 5 ms → high
3. Wait for startup (tens of ms)
4. SCCB read CHIP_ID (verify communication)
5. Load the initial register configuration (vendor-supplied)
6. Configure output format, resolution, frame rate
7. Configure auto controls (AE / AWB)
8. Configure image processing (sharpening, noise reduction)
9. Start video stream

## 9. Data Format and Units

- **RGB Bayer**: 1 color component per pixel (R/Gr/Gb/B); requires ISP processing
- **YUV422**: luma Y + chroma U/V; saves bandwidth
- **RGB565**: 16 bit per pixel (5+6+5)
- **JPEG**: compressed image stream
- **H.264 / H.265**: video compression
- **NV12 / NV21**: YUV420 memory layout
- Bandwidth: 1080p RGB565 60 fps ≈ 248 MB/s


## 10. Debugging Methods

1. Test SCCB communication
2. Read CHIP_ID to verify
3. Output a color-bar pattern (built-in test pattern)
4. Adjust lens focus to confirm sharpness
5. Check exposure and white balance
6. Measure the frame rate (data stream rate)
7. Check data-line signal integrity (scope)


## 11. Frequently Asked Questions

- **Color cast**: white balance not calibrated
- **Over / under exposure**: exposure parameters wrong
- **MIPI data errors**: differential line impedance mismatch, length mismatch
- **Wrong resolution output**: wrong register configuration
- **Choosing global shutter**: MT9V034, OV9281 (rolling shutter causes distortion on fast motion)
- **Low-light noise**: use a large-pixel sensor (IMX477)

## 13. Auto Control (AE / AWB / AF)

- **AE (Auto Exposure)**: adjust exposure time and gain based on scene lighting
- **AWB (Auto White Balance)**: adjust R/B gain based on scene color temperature
- **AF (Auto Focus)**: use VCM (voice coil motor) to move the lens
- Disable auto control: manual exposure suits industrial inspection
- Trigger mode: hardware trigger to synchronize flash


## 14. Image Processing

- **ISP (Image Signal Processor)**: Bayer → RGB, noise reduction, HDR, sharpening
- **3A algorithms**: AE / AWB / AF control
- **Distortion correction**: fisheye-lens distortion
- **Image encoding**: H.264 / H.265 hardware encoding
