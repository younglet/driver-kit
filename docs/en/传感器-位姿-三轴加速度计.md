# 3-Axis Accelerometer


## 1. Module Overview

A 3-axis accelerometer measures acceleration (including the gravity component) along the X, Y, and Z axes. It is a MEMS motion sensor used for orientation detection, step counting, fall detection, vibration monitoring, screen rotation, tap recognition, motion wake-up, and similar use cases.

The accelerometer measures specific force — i.e. the acceleration vector minus gravity. When the device is stationary, the output corresponds to the gravity component projected onto each axis; during motion, the motion-acceleration component is added.


## 2. Working Principle and Use Cases

The core is a MEMS (micro-electromechanical systems) structure: a small proof mass is suspended over a silicon substrate by flexible beams. Acceleration displaces the mass, and capacitive, piezoresistive, or piezoelectric elements detect that displacement and convert it into a voltage.

Typical applications:
- Auto screen rotation on phones and tablets
- Step counting, sleep monitoring, fall detection on wearables
- Attitude estimation for drones and robots (fused with gyroscope)
- Industrial vibration monitoring and shock detection
- Hard-drive drop protection

## 5. Hardware Connection

- Power: 1.8V~3.6V; add a 100nF decoupling capacitor close to the VDD pin
- CS pin: selects I2C/SPI mode (ADXL345: high = SPI, low = I2C)
- I2C pull-up: 4.7kΩ pull-ups to VDD on SDA/SCL
- INT1/INT2: interrupt outputs, configurable as push-pull or open-drain
- 5V tolerance: most modules include an LDO regulator to 3.3V, but IO is typically 3.3V
- Mounting orientation: follow the chip datasheet markings; mark X/Y arrows on the PCB
- Layout: keep sensitive analog sections away from switching power and motor drive traces


## 6. Communication Method

- I2C: standard 100/400 kHz; some support 1 MHz / 3.4 MHz
  - ADXL345 default address 0x53 (SDO=high) or 0x1D (SDO=low)
  - LIS2DH default 0x19 (SA0=high) or 0x18 (SA0=low)
- SPI: 4-wire; CPOL/CPHA mode 3 or 0 (check datasheet)
- Interrupts: level or pulse, high or low active, configurable


## 7. Initialization Flow

1. Configure the I2C/SPI bus (set speed and pins)
2. Read the `WHO_AM_I` register to verify the device
3. Soft-reset and wait for boot completion
4. Set the range (`±4g` / `±8g` / `±16g`)
5. Set the output data rate (ODR)
6. Configure FIFO mode (bypass / stream / trigger / trigger-stream)
7. Configure interrupt mapping (data-ready, free-fall, motion, etc.)
8. Clear all interrupt flags
9. Enable measurement mode

## 9. Data Format and Units

- Conversion: acceleration = raw value / sensitivity (e.g. 16-bit at ±2g: sensitivity = 16384 LSB/g)
- Units: commonly `g` (9.80665 m/s²) or `m/s²`
- Update timing: determined by ODR (e.g. 100 Hz = every 10 ms)
- Byte order: MSB first
- Timing constraint: reading a full 6-byte X/Y/Z frame should finish within one ODR period


## 10. Calibration and Filtering

1. Collect N samples (N ≥ 500 recommended) while the device is stationary
2. Compute the mean per axis as the offset
3. Store the offset with a version number and CRC32
4. Recalibrate when the temperature changes by more than 10°C
5. 6-sided method (for high-precision needs): record data with ±X, ±Y, ±Z facing down to build a correction matrix and offset
6. Corrected value = (raw - offset) × scale


## 11. Interrupts and DMA

- Data-ready interrupt (once per frame)
- FIFO: bypass / stream / FIFO / trigger / trigger-stream
- Wake-up interrupt (motion → wake host from sleep)
- Interrupt pins: INT1/INT2 routing with independent level / open-drain configuration
- Overflow flags: watermark / overflow; must be read in time


## 12. Error Handling

- No-ACK on bus: check address, pull-ups, power
- WHO_AM_I mismatch: counterfeit chip or wrong address
- Data overflow: range or ODR exceeds physical limits
- Long-term drift: temperature change or mechanical shock shifting the offset
- FIFO overflow: lower ODR or read more frequently
- Recovery flow: retry → soft-reset → re-initialize → report fault


## 13. Low Power

- Operating modes: Normal / Low Power 1/2/3/4 (different ODR tiers)
- Sleep current: under 1 µA
- Wake sources: motion / FIFO watermark / tap / free-fall
- Registers usually need to be reconfigured after wake-up

## 15. Debugging Methods

1. Read `WHO_AM_I` to confirm the model
2. Read while stationary; Z ≈ 9.8 m/s², X/Y ≈ 0
3. Flip the module and confirm each axis follows the right-hand rule
4. Observe the INT pin on a scope; data-ready frequency should equal ODR
5. Shake the module and confirm the waveform tracks the motion
6. Long-term monitor the bias stability
7. Monitor output drift when the temperature changes


## 16. Frequently Asked Questions

- **Z axis is not 9.8 m/s² when stationary**: offset not calibrated, or mounting tilt
- **Module is hot**: power reversed or short circuit
- **Cannot read over I2C**: wrong address, missing pull-ups, CS pin not pulled low (ADXL345)
- **Severe data jumps**: mechanical vibration not isolated, noisy power rail
- **Range saturation**: motion exceeds range — increase range or reduce motion
