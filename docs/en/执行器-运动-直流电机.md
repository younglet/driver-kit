# DC Motor


## 1. Module Overview

A DC motor is driven by a DC voltage to rotate, and is the most common rotary actuator. It is widely used in:

- Smart car drive wheels
- Toys
- Power tools (electric drills, fans)
- Home appliances (washing machines, vacuum cleaners)
- Fans and pumps
- Robot joints (requires encoder-based closed loop)

Types:

- **Brushed DC (BDC)**: classic, low cost, brush wear
- **Brushless DC (BLDC)**: electronic commutation, long life, high efficiency (see Brushless Motor doc)
- **Coreless motor**: high speed, ironless, low inertia
- **Cup motor**: ultra-small
- **Servo motor**: integrated encoder closed loop (see Servo Motor doc)


## 2. Working Principle and Use Cases

A current-carrying wire experiences a force in a magnetic field (left-hand rule). Brushes and a commutator automatically switch the armature polarity so the rotor keeps spinning.

Speed formula:

```text
n = (U - I × Ra) / k_e
U = supply voltage
I = armature current
Ra = armature resistance
k_e = back-EMF constant
```

Speed control:

- **PWM speed control**: vary the average voltage (duty cycle)
- **H-bridge commutation**: change rotation direction
- **Series resistor**: simple but inefficient
- **Voltage drop**: linear regulation, low efficiency

**Selection suggestions**:

- Car drive: JGA25-370, RS-555, TT motor
- Toys/makers: N20, TT motor, 130 motor
- High torque: GA12-N20 with encoder + gearbox
- Low noise, long life: BLDC (see Brushless Motor doc)
- Industrial control: Servo Motor (see Servo Motor doc)


## 5. Hardware Connection

- H-bridge driver: must use an H-bridge driver chip (e.g., TB6612, L298N, BTS7960, DRV8871)
- **Do not wire the H-bridge directly to MCU pins!** Direct connection will short the supply
- Flyback diodes: built into driver chips, but add TVS/Schottky externally
- Decoupling capacitors: 100nF + 100µF
- Heat dissipation: high-current motors need a heatsink
- Reverse EMF: sudden stop creates high-voltage spikes; needs a flyback path
- PWM frequency: 1 kHz ~ 25 kHz typical
- Dead time: avoid H-bridge shoot-through
- Encoder: closed-loop control requires AB quadrature encoder


## 6. Communication Method

- **PWM signal**: speed control (duty 0~100%)
- **H-bridge direction**: two GPIO lines (IN1/IN2 or PWM + DIR)
- **Analog voltage**: some driver boards (e.g., BTS7960) support analog speed control
- **Encoder feedback**: AB quadrature pulses


## 7. Initialization Flow

1. Configure GPIO (IN1/IN2/PWM outputs)
2. Configure PWM module (20 kHz)
3. Initialize to stopped state (IN1=IN2=0)
4. Configure encoder inputs (if used)
5. Set PID parameters (closed loop)
6. Start


## 8. Driver Interface

```python
class DCMotor:
    def __init__(self, in1, in2, pwm):
        self.in1 = in1
        self.in2 = in2
        self.pwm = pwm
        self.encoder = None

    def begin(self):
        pass

    def set_speed(self, percent):
        """-100~+100, direction and speed."""
        if percent > 0:
            self.in1.high(); self.in2.low()
            self.pwm.duty(percent)
        elif percent < 0:
            self.in1.low(); self.in2.high()
            self.pwm.duty(-percent)
        else:
            self.in1.low(); self.in2.low()
            self.pwm.duty(0)

    def stop(self):
        self.in1.low(); self.in2.low()
        self.pwm.duty(0)

    def brake(self):
        """Short-circuit braking."""
        self.in1.high(); self.in2.high()
        self.pwm.duty(100)

    def set_encoder(self, encoder):
        self.encoder = encoder

    def velocity(self):
        """Returns rpm."""
        if self.encoder:
            return self.encoder.read_velocity()
        return 0
```



```python
def pid_control(target_rpm, current_rpm, kp, ki, kd, integral, prev_err):
    err = target_rpm - current_rpm
    integral += err
    derivative = err - prev_err
    output = kp * err + ki * integral + kd * derivative
    return output, integral, err
```

Typical PID parameters: kp=0.5, ki=0.1, kd=0.05 (tune on site).


## 9. Data Format, Units, and Timing

- PWM duty cycle: 0~100% (or 0~65535)
- Speed: rpm or deg/s
- Current: A
- Torque: N·m
- Encoder count: pulses


## 10. Calibration, Compensation, and Filtering

- Encoder data: median filter to remove spikes
- PWM output: hardware filtering (motor coil itself)
- Current sampling: RC low-pass filter


## 11. Interrupts, DMA, FIFO, and Buffers

- Encoder interrupt: capture A/B signals
- Timer interrupt: PID period (e.g., 1 ms)
- DMA: encoder counting (reduces CPU load)


## 12. Error Handling and Exception Recovery

- **Stall**: current spikes sharply; triggers over-current protection
- **Over-voltage**: reverse-EMF spikes; add flyback path
- **Overheating**: exceeds rated current; add cooling or reduce power
- **Lost steps**: closed-loop system PID misconfigured
- **Encoder failure**: fallback value or software reset


## 13. Low Power

- PWM speed control is more efficient than resistor drop
- Stop the motor when idle (0% PWM)
- Slow down briefly to save energy


## 14. Example Code

```python
class DCMotor:
    def __init__(self, in1, in2, pwm):
        self.in1 = in1
        self.in2 = in2
        self.pwm = pwm

    def begin(self):
        self.stop()

    def forward(self, speed):
        """speed 0~100."""
        self.in1.high()
        self.in2.low()
        self.pwm.duty(speed)

    def backward(self, speed):
        self.in1.low()
        self.in2.high()
        self.pwm.duty(speed)

    def stop(self):
        self.in1.low()
        self.in2.low()
        self.pwm.duty(0)
```

## 15. Debugging Methods

1. Slowly increase PWM duty cycle and observe motor start
2. Test forward/reverse switching
3. Measure no-load current and stall current
4. PID tuning: start with P, then add I/D
5. Long-time run to monitor temperature


## 16. Frequently Asked Questions

- **Motor does not turn**: power not connected, H-bridge logic wrong, PWM not output
- **PWM but no rotation**: H-bridge dead time too long, insufficient current
- **Jitter**: PID parameters wrong or encoder signal noisy
- **Overheating**: prolonged stall or over-rated current
- **Reverses but no speed control**: PWM frequency too low (e.g., 100 Hz causes motor whining)
