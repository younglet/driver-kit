# Laser Distance Sensor (ToF)


## 1. Module Overview

A laser distance sensor (ToF, Time-of-Flight) measures the light's flight time or counts photons to compute distance. It offers high accuracy (mm-level), fast response, and a moderate range.

Types:
- **Direct ToF**: measures light flight time (nanosecond scale)
- **Indirect ToF (iToF)**: measures phase shift
- **Laser phase ranging**: modulated laser + phase-difference measurement (tens of meters)
- **Laser pulse ranging**: high-peak laser pulses for long range (kilometers)
- **LIDAR**: scanning laser ranging

Applications: robot obstacle avoidance, drone altitude hold, robot vacuum mapping, AGV, smart locks, gesture recognition, phone focus assist.


## 2. Working Principle and Use Cases

### 2.1 ToF Principle

```text
distance = (speed of light × flight time) / 2
speed of light ≈ 3×10⁸ m/s
```

Light needs 3.33 ns to travel 1 meter. 1 ns resolution corresponds to 15 cm, so iToF (phase method) is more common.

### 2.2 iToF

Emits modulated light (10~100 MHz), measures the phase shift after reflection. Phase shift → distance.

### 2.3 Laser Pulse Ranging

Emits high-peak pulses (tens of watts); an APD (avalanche photodiode) receives them; single-photon detection (SPAD).

## 4. Key Parameters and Selection Guide

| Parameter | Description |
|---|---|
| Range | 1 mm ~ 350 m (model-dependent) |
| Accuracy | 1~3 mm (ToF) / several mm ~ 1 cm (pulse) |
| Resolution | mm-level |
| Field of view | Single point ~ multi-zone |
| Measurement rate | 1~50 Hz (ToF) / 1~10 kHz (pulse) |
| Interface | I2C / UART / SPI |
| Supply voltage | 3.3V / 5V |
| Operating current | 20~200 mA (model-dependent) |
| Operating temperature | -20~+70°C |
| Sunlight resistance | Significant indoor vs outdoor difference |

**Selection suggestions**:
- Short range (mm~10 cm): VL6180X, VL53L4CD
- Medium range (meter): VL53L1X, TFMini
- Multi-zone: VL53L5CX / VL53L7CX
- Long range (10~100 m): TFMini-S, Benewake TF series
- 2D scanning: RPLIDAR, YDLIDAR
- Industrial grade: Northbound, LeiShen Intelligent, etc.


## 5. Hardware Connection

- I2C pull-ups: 4.7kΩ
- VL53L series XSHUT pin: used to power down the sensor and change its I2C address (multi-device timing control)
- TFMini: 5V supply, 3.3V signals
- Anti-glare: avoid direct sunlight on the receiver window
- Dust / waterproof: add an enclosure for outdoor use
- Multiple ToFs used simultaneously will interfere with each other — stagger the measurement timing


## 6. Communication Method

- I2C: VL53L series, TFMini-S, TOF10120
- UART: TFMini, TOF10120, LiDAR
- USB: RPLIDAR, etc.
- Measurement data: distance + strength (confidence)
- Multi-zone ToF: array of zone distances (e.g. 8x8 = 64 points)


## 7. Initialization Flow

1. Configure I2C (default address 0x29, can be changed to 0x2A, etc.)
2. Check sensor ID (0xEACC)
3. Soft start (XSHUT pulled high)
4. Configure measurement mode (short / medium / long range)
5. Set ROI (region of interest)
6. Set measurement timing budget
7. Start measurement
8. Wait for data-ready
9. Read distance + range status + signal strength

## 9. Data Format and Units

- Distance unit: mm (VL53L) / cm (TFMini) / m (long range)
- Status code: 0 = valid, 1-7 = error (VL53L1)
- Signal strength (SIG/SPAD): the higher, the more reliable
- Ambient light intensity


## 10. Calibration and Filtering

- VL53L series support SPAD and offset calibration (Xtalk calibration)
- Factory calibration can be stored in NVM
- Distance vs actual deviation: cover-window glass requires compensation

- Median filter (removes spikes)
- Moving average (reduces jitter)
- Range status check (VL53L): data is unreliable when status is non-zero


## 11. Interrupts and DMA

- Data-ready interrupt (GPIO INT)
- Multi-ToF timing control (stagger measurements)
- Some chips support FIFO


## 12. Error Handling

- Distance saturation (out of range): returns 8190 or maximum value
- Weak signal (< 50): may be outside the range
- Repeated measurement failures: check I2C and sensor hardware
- Interference: multiple ToFs used simultaneously, sunlight


## 13. Low Power

- VL53L1X: measurement ~20 mA, standby ~5 µA
- Intermittent measurement (100 ms interval) is sufficient for most applications
- TFMini: measurement ~140 mA, standby ~10 mA

## 15. Debugging Methods

1. Test against white paper / wall at 10~200 cm
2. Test different colors (black, white, red)
3. Place glass in front and observe attenuation
4. Check reading stability in sunlight
5. Multi-ToF interference test


## 16. Frequently Asked Questions

- **VL53L default address conflict**: 0x29; XSHUT controls address change
- **Distance reads too small / too large**: cover glass thickness not compensated
- **High measurement failure rate**: sunlight interference, target outside range
- **TFMini unstable**: check UART wiring and baud rate
- **Multiple ToFs interfering with each other**: stagger the measurement timing
