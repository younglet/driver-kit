# Motor Driver


## 1. Module Overview

A motor driver is the power electronics device connecting the MCU to the motor. It converts the MCU's low-power control signal into the high-power signal needed to drive the motor. Common types:

- **H-bridge driver**: for DC motors (BDC)
- **Three-phase driver**: for brushless motors (BLDC) and permanent-magnet synchronous motors (PMSM)
- **Stepper driver**: for stepper motors (see Stepper Motor doc)
- **Servo driver**: for servo motors (see Servo Motor doc)

This article focuses on general H-bridge and three-phase BLDC/PMSM drivers.


## 2. Working Principle and Use Cases

### 2.1 H-bridge

Four switches (MOSFET / IGBT) form an H shape with the motor in the middle. By controlling the switch combinations:

- Forward: upper-left + lower-right
- Reverse: upper-right + lower-left
- Brake: both upper or both lower on same side
- Coast: all off

PWM speed control: insert a PWM duty cycle when switching to control the average voltage.

### 2.2 Three-phase Driver

Three half-bridges (6 switches) control the three-phase motor windings. The energizing sequence switches according to rotor position:

- 6-step trapezoidal
- FOC (field-oriented control) sinusoidal
- Needs driver chips (e.g., DRV8301, FD6288)

### 2.3 Key Parameters

- Dead time: switching interval between upper and lower arms (prevents shoot-through)
- PWM frequency: 1 kHz ~ 100 kHz
- Current sensing: low-side / high-side
- Back-EMF / current feedback: for closed-loop control

## 4. Key Parameters and Selection Guide

| Parameter | Description |
|---|---|
| Working voltage | V (max) |
| Output current | A (continuous / peak) |
| Rds(on) | on-resistance (mΩ), lower is more efficient |
| PWM frequency | kHz |
| Dead time | ns / µs |
| Logic voltage | 3.3V / 5V |
| Interface | PWM + DIR / SPI / I2C / analog |
| Protection | over-current, over-temperature, under-voltage, short circuit |
| Heat dissipation | heatsink / fan required |
| Package | SOIC / QFN / module |

**Selection suggestions**:

- Maker / car: TB6612, L298N, DRV8871
- Industrial high-current: BTS7960, VNH2SP30, Cytron MD30C
- BLDC: DRV8301 + MOSFET module
- FOC: DRV8323 + MCU (STM32 / ESP32)
- Smart BLDC: ODrive, VESC
- Micro: STSPIN233, HR8833


## 5. Hardware Connection

- Power supply: clean, regulated, sufficient current
- Decoupling capacitors: 100nF + 100µF on motor supply
- Bootstrap capacitor: bootstrap-type half-bridge drivers (e.g., IR2104) need a 100nF bootstrap capacitor
- Flyback diodes: built-in or external (depending on part)
- Heat dissipation: high-current applications must mount a heatsink
- **Do not connect the H-bridge directly to MCU pins**: confirm power and ground are wired correctly before starting
- Encoder interface: BLDC FOC requires AB encoder (some support sensorless)
- Control signals: logic-level compatible (3.3V / 5V)
- **Dead time must be sufficient**: otherwise shoot-through, instant burn-out



- RθJA and power dissipation determine temperature rise
- PCB copper pour for heat dissipation (exposed copper)
- Heatsink (aluminum)
- Fan (forced air cooling)
- Temperature-rise test: measure case temperature under actual load (< 80°C)


## 6. Communication Method

- **PWM + DIR**: simple (most H-bridges)
- **SPI**: smart driver register configuration
- **I2C**: some compact drivers
- **Analog voltage**: a few H-bridges
- **CAN / UART**: smart drivers (VESC, ODrive)


## 7. Initialization Flow

### 7.1 H-bridge

1. Configure MCU GPIO (PWM + DIR)
2. Set initial state to stopped (PWM = 0)
3. Slowly raise PWM at startup

### 7.2 BLDC Driver

1. Configure pre-driver enable
2. Configure gate-drive current
3. Configure current sensing (shunt resistor)
4. Configure protection (over-current threshold)
5. Configure PWM frequency and dead time
6. Enable output

### 7.3 FOC Driver

1. Measure motor parameters (resistance, inductance, pole pairs)
2. Configure encoder
3. Set control parameters (PID)
4. Calibrate current and angle offset
5. Start


## 8. Driver Interface

```python
class HBridge:
    def __init__(self, pwm, dir_pin):
        self.pwm = pwm
        self.dir = dir_pin

    def begin(self):
        pass

    def forward(self, speed):
        self.dir.low()
        self.pwm.duty(speed)

    def backward(self, speed):
        self.dir.high()
        self.pwm.duty(speed)

    def brake(self):
        self.pwm.duty(100)  # full duty short-circuit

    def coast(self):
        self.pwm.duty(0)

## 9. Data Format, Units, and Timing

- PWM: duty cycle 0~100% or 0~65535
- SPI: register configuration
- Analog: voltage V
- FOC: dq-axis current


## 10. Error Handling and Exception Recovery

- Over-current protection (OCP): exceeds threshold, shuts down output
- Over-temperature protection (OTP): exceeds temperature limit, shuts down
- Under-voltage protection (UVP): low supply, shuts down
- Short-circuit protection: shoot-through immediately shuts down
- Reverse-EMF protection: use flyback path
- PWM-fail protection: signal-line break shuts down



- **Driver alarm**: read alarm register / LED blink
- **Motor doesn't turn**: PWM not output, H-bridge not enabled, power not on
- **Severe heating**: over-current, insufficient heat dissipation
- **H-bridge burnt out**: insufficient dead time, power short circuit
- **Encoder wrong**: FOC driver cannot commutate


## 11. Low Power

- Quiescent current < 100 µA (most)
- Disable to reduce static consumption
- Standby mode (some smart drivers)


## 12. Example Code

```


## 13. Debugging Methods

1. Without motor, measure driver output waveform
2. Low PWM to start the motor (10%) and observe
3. Measure current waveform (oscilloscope + current probe)
4. Test forward / reverse
5. Measure temperature during long-time operation
6. Test over-current protection


## 14. Frequently Asked Questions

- **Motor doesn't turn**: enable not on, signal not output, power not on
- **Excessive current**: mechanical jam, acceleration too fast
- **Driver overheating**: insufficient cooling, over-current
- **H-bridge burnt out**: insufficient dead time, shoot-through
- **FOC start fails**: motor parameters inaccurate, encoder wiring wrong
- **Closed-loop oscillation**: PID misconfigured, noisy current sense
