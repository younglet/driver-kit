# Infrared Distance Sensor


## 1. Module Overview

An infrared distance sensor emits modulated IR light and uses the reflected light intensity or angle to estimate distance. Common uses:
- Robot obstacle avoidance
- Object proximity detection
- Automatic door sensing
- Liquid-level detection
- Black-line tracking
- Industrial product counting

Types:
- **Triangulation (Sharp GP2Y series)**: PSD position-sensitive detector, analog voltage output
- **Reflective intensity**: IR emitter + receiver pair
- **ToF**: see `Laser Distance.md`
- **Reflective photo-interrupter**: emitter-detector pair, single-point detection


## 2. Working Principle and Use Cases

### 2.1 Triangulation (Sharp)

An IR LED emits a beam toward the target; the reflected light passes through a lens onto a PSD (position-sensitive detector). Different distances produce different reflection positions; distance is calculated via trigonometry. The output vs distance curve is non-linear (steep near, flat far).

### 2.2 Reflective Intensity

IR emission → reflected to photodiode → signal strength depends on distance and reflectivity.
- Dark objects reflect weakly → reported distance appears larger
- Bright objects reflect strongly → reported distance appears closer
- Output is non-linear; precise ranging is difficult

## 5. Hardware Connection

- Analog output: ADC sampling; note the voltage range (Sharp output 0.4V~3.1V)
- Digital output: GPIO reading the high/low level
- I2C: GP2Y0E series default 0x40
- Reject ambient light: Sharp uses internal modulation (modulation frequency 25 kHz)
- Note: sensitive to sunlight and IR (facing direct sunlight will saturate)
- Reflective surface: white is best; black has large error
- Mounting: face the object directly; tilt angle < 5°


## 6. Communication Method

- Analog: voltage output
- Digital: high / low level (threshold set by hardware comparator or MCU code)
- I2C: GP2Y0E series supported (GP2Y0E02/03)
- GP2Y0D series: digital TTL output with fixed threshold


## 7. Initialization Flow

1. Configure ADC (analog) or GPIO / I2C (digital)
2. Power-on settling (~30 ms)
3. Periodic sampling
4. Digital output read directly
5. Analog output converted via lookup or fitted curve

    """v 0.0~3.1 V → cm"""

## 9. Data Format and Units

- Analog output: voltage (V)
- Distance: cm
- Digital output: high / low level
- Unit conversion: via lookup table or fitted formula


## 10. Calibration and Filtering

Sharp series is non-linear; typical curve:
- 10 cm → 2.3 V
- 20 cm → 1.1 V
- 50 cm → 0.5 V
- 80 cm → 0.4 V

Common engineering formula:

```python
# GP2Y0A21YK0F
def gp2y0a21_21(v):
    if v <= 0.0:
        return 80.0
    return 29.98 / (v + 0.1) - 5.0
```

- Median filter to remove spikes
- Moving average to reduce jitter
- Skip voltages below 0.4 V (long-range saturation)
- Skip voltages above 3.1 V (blind zone)


## 11. Interrupts and DMA

- Digital output can be used as an interrupt / wake-up source
- Analog sampling period of 30~50 ms is enough


## 12. Error Handling

- Distance saturation: long range or strong reflection (sunlight outdoors)
- Unstable digital output: fuzzy threshold — use hardware comparator or software filtering
- Distance skewed: difference in reflectivity; use relative position rather than absolute distance
- Analog reads 0: power not applied


## 13. Low Power

- Sharp module draws ~30 mA
- Intermittent measurement: 1 Hz is enough for most applications
- Digital output modules can lower the rate when no target is detected

## 15. Debugging Methods

1. Test against a white wall at various distances between 10~80 cm
2. Compare readings between black and white objects
3. Test in direct sunlight (watch for saturation)
4. Inspect the waveform (modulated light at 25 kHz)


## 16. Frequently Asked Questions

- **Sharp output near 0.4 V**: long range or no target
- **Digital output toggles frequently**: unstable threshold; use a hardware comparator or software filter
- **Distance reads too small / too large**: reflectivity difference; prefer relative position over absolute distance
- **Saturates under sunlight**: add a light shield
- **Which ToF to pick**: for higher precision, see ToF (VL53L series)
