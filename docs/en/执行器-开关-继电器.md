# Relay


## 1. Module Overview

A relay is an electronic switch that uses a small current to control a large current. Common uses:

- Smart home (lighting, appliance control)
- Industrial automation (PLC)
- Power system protection
- Automotive electronics
- Security systems
- Remote control
- Power switching
- Starting high-power motors

Types:

- **Electromagnetic Relay (EMR)**: mechanical contacts, low cost
- **Solid-State Relay (SSR)**: no contacts, long life
- **Latching Relay**: pulse-triggered, holds state, energy-saving
- **Reed Relay**: fast switching
- **Power Relay**: high current
- **Signal Relay**: low-level signal switching


## 2. Working Principle and Use Cases

### 2.1 Electromagnetic Relay

When the coil is energized, it produces a magnetic field that attracts the armature and moves the contacts. The spring resets the armature when power is removed.

### 2.2 Solid-State Relay

The input signal controls a semiconductor switch (thyristor / MOSFET) to achieve "contactless" switching.

### 2.3 Latching Relay

A pulse-driven coil changes the state, and a mechanical latch holds it. No continuous power is required.

## 5. Hardware Connection

- **Flyback diode**: reverse-parallel 1N4148 or 1N4007 across the coil
- **Drive current**: 5V relay coil ~70 mA; MCU direct drive is limited — use a transistor (e.g., S8050) to boost
- **Optocoupler isolation**: modular versions usually include optocouplers, isolating MCU from the relay
- **Contact protection**: inductive loads (motors, coils) need RC snubber or MOV
- **Arcing**: high-voltage / high-current loads show obvious arcing on contacts, which shortens life
- **Heat dissipation**: SSRs at high current need a heatsink (> 30A)
- **Mechanical life**: EMRs have limited life; consider SSRs for frequent switching



Inductive loads (motors, transformers, coils) generate back-EMF when disconnected and require protection:

- **RC snubber**: 100Ω + 0.01µF in series, parallel across the contacts
- **Varistor (MOV)**: clamps voltage
- **Flyback diode**: for DC coils
- **Gas discharge tube**: for high-voltage protection


## 6. Communication Method

- **GPIO**: direct control (most common)
- **I2C / SPI**: via expander chips (PCF8574, MCP23017)
- **RS485 / Modbus**: industrial bus control
- **WiFi / BLE**: smart relays
- **IR remote**: smart home


## 7. Initialization Flow

1. Configure GPIO (output mode)
2. Set initial state to OFF
3. Start control after power is stable
4. Check heat dissipation at high current


## 8. Driver Interface

```python
class Relay:
    def __init__(self, pin, active_high=True):
        self.pin = pin
        self.active_high = active_high

    def begin(self):
        self.off()

    def on(self):
        if self.active_high:
            self.pin.high()
        else:
            self.pin.low()

    def off(self):
        if self.active_high:
            self.pin.low()
        else:
            self.pin.high()

    def toggle(self):
        self.pin.toggle()


class SSRelay(Relay):

    pass


class LatchingRelay:
    """Latching relay."""
    def __init__(self, set_pin, reset_pin):
        self.set_pin = set_pin
        self.reset_pin = reset_pin

    def begin(self):
        pass

    def latch(self):
        """Pulse trigger to close."""
        self.set_pin.high()
        time.sleep_ms(50)
        self.set_pin.low()

    def unlatch(self):
        self.reset_pin.high()
        time.sleep_ms(50)
        self.reset_pin.low()
```


## 9. Data Format, Units, and Timing

- Control signal: boolean (on / off)
- Bus control: register address + on/off bit
- Modbus: coil register (function codes 0x01 / 0x05 / 0x0F)


## 10. Error Handling and Exception Recovery

- **Contact welding**: overload or severe arcing — replace the relay
- **No response**: coil not energized, insufficient drive current
- **Contact oxidation**: fails with low-current loads — use "wet circuit" or gold-contact versions
- **SSR failure**: over-current or insufficient heat dissipation, burnt out
- **Bus communication error**: wrong Modbus address or baud rate



- High voltage is dangerous: 220V AC work requires professional handling
- Electric shock protection: insulating gloves and tools
- Fire risk: welded contacts + heavy load may ignite


## 11. Low Power

- Electromagnetic relay: coil consumes power continuously when energized
- Latching relay: holds state after pulse, zero power
- SSR: control side 5~20 mA


## 12. Example Code

```python
relay = Relay(pin, active_high=True)
relay.begin()
relay.on()
time.sleep_ms(5000)
relay.off()

# Multiple relays
relays = [Relay(p1), Relay(p2), Relay(p3)]
for r in relays:
    r.begin()

# Toggle
for i in range(2):
    relays[i].on()
    relays[(i+1) % 3].off()
    time.sleep_ms(1000)
```


## 13. Debugging Methods

1. Listen for the relay's click
2. Measure the voltage across the contacts (220V when OFF / 0V when ON)
3. Check the coil voltage
4. Measure temperature during long operation
5. Measure load current


## 14. Frequently Asked Questions

- **Relay doesn't pull in**: insufficient coil voltage, insufficient drive current
- **Welded contacts**: overload or arcing
- **Contacts don't conduct**: contact oxidation (low-current failure)
- **SSR doesn't work**: DC SSR wired to AC or vice versa
- **Insufficient cooling**: SSR burnt out at high current
- **Coil burnt out**: missing flyback diode, back-EMF breakdown
- **Bus module communication failure**: wrong address or baud rate

## 16. Contact Life

- Resistive load: long electrical life
- Inductive load: severe arcing, electrical life drops sharply
- Capacitive load (LED lamps, capacitors): high inrush current, requires surge protection
- Motor load: 5~10× inrush current, requires derating
