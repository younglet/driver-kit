# Magnetometer


## 1. Module Overview

A magnetometer measures the three-dimensional magnetic-field vector. It is commonly used for electronic compasses, attitude estimation, magnetic-field sensing, geomagnetic navigation, and position detection.

A standalone magnetometer can provide absolute heading (magnetic north). Fused with an accelerometer / gyroscope, it yields a stable attitude (see the 9-axis IMU doc).


## 2. Working Principle and Use Cases

Based on the Hall effect or anisotropic magnetoresistance (AMR) / giant magnetoresistance (GMR) / tunneling magnetoresistance (TMR). Earth's magnetic field is roughly 25~65 µT, so the magnetometer needs µT-level range and high resolution to measure the geomagnetic vector.

Applications:
- Phone / tablet electronic compass
- Drone heading assistance
- AR/VR head orientation
- Indoor magnetic positioning (iBeacon-like)
- Door/window switch detection (with a magnet)
- Motor current sensing (indirect)

## 5. Hardware Connection

- Keep away from interference: motors, batteries, speakers, ferromagnetic materials (including screws and shields)
- Module placement: stay away from high-current PCB traces (DC-DC switching currents couple into the magnetic field)
- I2C pull-ups: 4.7kΩ
- Most chips run at 1.8V / 3.3V
- Stacking: some 9-axis modules place the magnetometer on an independent sub-board to avoid main-PCB interference


## 6. Communication Method

- I2C: standard 100 / 400 kHz
  - HMC5883L default address 0x1E
  - QMC5883L default address 0x0D
  - LIS3MDL default 0x1C (SDO=high) or 0x1E (SDO=low)
  - AK09911 default 0x0C
  - AK09918 default 0x0C
- SPI: supported on some models
- Single-axis analog: read directly via ADC


## 7. Initialization Flow

1. Verify WHO_AM_I
2. Soft-reset
3. Set range, ODR, and bandwidth
4. Set measurement mode (single / continuous)
5. Trigger measurement (e.g. HMC5883L requires writing 0x01 to the Mode register)
6. Read X/Y/Z data

## 9. Data Format and Units

- Unit: µT (Tesla) or mGauss (milli-Gauss); 1 µT = 10 mGauss
- Byte order: MSB first
- Data update: single mode requires manual trigger; continuous mode updates at ODR
- Range vs sensitivity: Gauss → µT, 1 G = 100 µT
- Geomagnetic reference: horizontal component ~ 20~50 µT depending on latitude
- Declination: angle between magnetic and true north (typically 0~10°, look up the local value)


## 10. Calibration and Filtering

### 10.1 Hard-iron Calibration

Constant ferromagnetic materials around the magnetometer cause a fixed offset. Rotate around the three axes and collect N samples; the ellipsoid center is the hard-iron offset:

```python
bx = (max_x + min_x) / 2
by = (max_y + min_y) / 2
bz = (max_z + min_z) / 2
```

### 10.2 Soft-iron Calibration

A non-uniform field distorts the ellipsoid; build a 3×3 correction matrix to pull the ellipsoid back into a sphere.

### 10.3 Simplified 9-point / ellipsoid fitting

Collect hundreds of samples while rotating, fit the ellipsoid, then derive the center and the transformation matrix.

## 11. Error Handling

- **Magnetic interference**: motors, batteries, power modules, magnets, speakers
- **Soft-start current surge**: DC-DC startup magnetic spikes
- **Temperature drift**: offsets at low/high temperatures
- **Magnetic anomaly**: saturation when too close to a magnet
- **Self-calibration failed**: user did not rotate enough

- WHO_AM_I error: check the I2C address (note HMC vs QMC incompatibility)
- All-zero data: measurement not triggered
- Data saturated: field too strong, too close to a magnet
- Bus stuck: I2C recovery


## 12. Low Power

- Single-shot mode: automatically enters low power after each measurement
- Measurement intervals can be as short as a few ms
- Most chips idle below 1 µA


## 13. Example Code

```python
class Magnetometer:
    def __init__(self, bus, address=0x0D):  # QMC5883L default
        self.bus = bus
        self.address = address
        self.bias = (0.0, 0.0, 0.0)

    def begin(self):
        pass

    def read_mag(self):
        """Return mx, my, mz in µT."""
        pass

    def calibrate(self, samples=1000):
        """Rotate around the three axes to collect samples and return the hard-iron offset."""
        pass

    def get_corrected_mag(self):
        mx, my, mz = self.read_mag()
        bx, by, bz = self.bias
        return mx - bx, my - by, mz - bz

    def heading(self, mx, my):
        """Heading = atan2(my, mx) * 180/π, plus declination correction."""
        import math
        return math.degrees(math.atan2(my, mx))


mag = Magnetometer(i2c, address=0x0D)
mag.begin()
mag.calibrate(samples=1000)
while True:
    mx, my, mz = mag.get_corrected_mag()
    deg = mag.heading(mx, my)
    print("mag=(%.1f,%.1f,%.1f) heading=%.1f°" % (mx, my, mz, deg))
```


## 14. Debugging Methods

1. Stationary readings should be stable
2. Rotate 90°/180°/270° and observe the heading change
3. Rotate in the horizontal plane → heading changes continuously
4. Flip the module → Z-axis reading changes
5. Move away from computers, phones, and speakers to test magnetic interference


## 15. Frequently Asked Questions

- **HMC5883L returns no data**: must write to the Mode register first to trigger a single measurement
- **Wrong QMC5883L address**: default 0x0D, not 0x1E — not compatible with HMC5883L
- **Magnetic heading offset**: declination not compensated
- **Heading points wrong**: module not level, or strong interference
- **Data overflow**: field too strong
