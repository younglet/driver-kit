# Storage Overview

Persistent storage for embedded systems. DriverKit covers **3 storage modules**, ranging from small parameter storage to large-capacity data.

## Module Quick Index

| Module | Description | Doc |
|---|---|---|
| EEPROM | Byte-level read/write, small parameters (1 Kbit ~ 1 Mbit) | [Open](./存储-非易失-EEPROM) |
| Flash | NOR large capacity, files / font libraries (1 Mbit ~ 1 Gbit+) | [Open](./存储-非易失-Flash) |
| SD Card | SDHC/SDXC, portable large capacity (2 GB ~ 1 TB+) | [Open](./存储-外部-SD卡) |

## Selection Quick Reference

| Application | Recommended |
|---|---|
| Device parameters (KB level) | [EEPROM](./存储-非易失-EEPROM) AT24C series |
| Frequent log writes | [EEPROM](./存储-非易失-EEPROM) FRAM FM24CL series |
| Chinese font library (MB level) | [Flash](./存储-非易失-Flash) W25Q series |
| File system (MB level) | [Flash](./存储-非易失-Flash) + LittleFS |
| Data logging (GB level) | [SD Card](./存储-外部-SD卡) |
| Video surveillance | [SD Card](./存储-外部-SD卡) high-speed (U3/V30) |
| Raspberry Pi system disk | [SD Card](./存储-外部-SD卡) A2 + 32 GB+ |

## Common Knowledge

- **EEPROM** is byte-writable, write endurance 10⁶ (FRAM unlimited)
- **Flash** operates by page/sector, erase-before-write, write endurance 10⁵
- **SD Card** uses PC-compatible file systems, but unsafe against sudden power loss
- **Power-loss protection**: CRC checksum, dual backup
- **Wear leveling**: LittleFS manages automatically; FAT needs application-level handling
