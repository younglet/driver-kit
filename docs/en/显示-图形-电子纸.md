# E-Paper (E-Ink)


## 1. Module Overview

E-Paper (E-Ink) is a low-power display technology that mimics the appearance of paper. Common use cases:

- E-book readers (Kindle);
- Electronic shelf labels (ESL);
- Logistics labels;
- Smart handwriting notebooks;
- Smart watches;
- Bus stop signs;
- Outdoor information boards;
- Instrument panels.

Features:

- Ultra-low power (0 power for static images);
- Paper-like appearance (readable in sunlight);
- 180° viewing angle;
- Relatively slow refresh (1~2 s for full refresh);
- Mostly black-and-white (color versions are less common);
- Bendable (flexible screens).


## 2. Working Principle and Use Cases

- **Electrophoretic display technology**: charged particles (black/white) suspended in a liquid move under an applied electric field;
- **Microcapsules**: each pixel is a microcapsule containing black/white particles;
- **Electric field control**: black particles move to the front = display black; white particles move to the front = display white;
- Black absorbs light, white reflects ambient light;
- **Bistable**: image is retained without continuous power.

Types:

- **Black-and-white EPD** (Electrophoretic Display): most common;
- **Three-color EPD** (B/W/R or B/W/Y): common for shelf labels;
- **Multi-color EPD**: ACeP (Advanced Color E-Paper) 4-color / 7-color;
- **Flexible EPD**: bendable.

## 5. Hardware Connection

- Digital power: 3.3V, decoupling capacitors;
- EPD high voltage: ±15V, +22V etc. (internal boost in driver IC);
- SPI: 4-wire SCK/MOSI/CS/DC;
- BUSY: EPD busy status output (important);
- RST: reset;
- ESD protection;
- Not bendable (except flexible screens);
- Temperature limit: refresh fails at low temperature;
- SPI speed: typical 10~20 MHz.


## 6. Communication Method

- **SPI**: 4-wire (SCK/MOSI/MISO/CS), DC command/data selection;
- **I2C**: some small screens support;
- Host: MCU (Arduino, ESP32, STM32, Raspberry Pi).


## 7. Initialization Flow

1. Configure GPIO (CS/DC/RST/BUSY);
2. Configure SPI;
3. Hardware reset (RST low → high);
4. Software reset;
5. Send driver initialization sequence (vendor provided);
6. Set display window (area);
7. Send pixel data;
8. Send refresh command;
9. Wait for BUSY to go low (refresh complete);
10. Enter sleep (save power).

### 10.1 Full Refresh

- Time: 1~4 s;
- Effect: no ghosting;
- Flicker: flashes multiple times quickly;
- Suitable for: first display, infrequent updates.

### 10.2 Partial Refresh

- Time: 0.3~1 s;
- Effect: may have ghosting (accumulated);
- Flicker: less flickering;
- Suitable for: frequent updates;
- Note: do one full refresh after every N partial refreshes to clear ghosting.

### 10.3 Fast Refresh

- Some screens support ultra-fast refresh (0.1~0.3 s);
- Severe ghosting, only suitable for short-term display.



- ASCII characters: 5x8 / 8x16 / 12x16;
- Chinese font library: requires Flash to store glyphs (GBK, Unicode);
- Bitmap: BMP monochrome conversion;
- LVGL supports vector fonts, but slow refresh limits practicality.



- Ghosting accumulates after multiple partial refreshes;
- Periodically do full refresh to clear ghosting (e.g. 1 full refresh every 10 partial refreshes);
- Some screens support auto ghosting removal.


## 9. Data Format, Units, and Timing

- 1 bit/pixel (B/W);
- B/W/R EPD: 1 bit B/W + 1 bit R (double buffer);
- ACeP: 1 byte/pixel (8 colors);
- Byte order: MSB first (row scan).


## 10. Interrupts, DMA, FIFO, and Buffers

- BUSY signal: high during refresh, low when complete;
- Interrupt: BUSY falling edge triggered.


## 11. Error Handling

- **BUSY always high**: SPI timing wrong, refresh command wrong;
- **Black screen**: power, reset wrong;
- **Severe ghosting**: no periodic full refresh;
- **Refresh failure**: temperature low (< 0°C);
- **Data error**: SPI speed too fast.


## 12. Low Power

Core advantages of E-Paper:

- Static display: 0 power (bistable);
- Power only consumed during refresh (20~40 mA·s);
- Enter sleep after refresh (< 10 µA);
- Battery-powered for years (with a few refreshes per day).


## 14. Debugging Methods

1. Check power voltage (including high voltage);
2. Measure BUSY signal timing;
3. Use oscilloscope to inspect SPI waveform;
4. Run vendor example code;
5. Test refresh at different temperatures;
6. Check ghosting situation.


## 15. Frequently Asked Questions

- **Black screen**: power, reset, initialization failed;
- **Severe ghosting**: no periodic full refresh;
- **Partial refresh failure**: some screens don't support it;
- **Refresh fails at low temperature**: temperature out of range;
- **Partial display**: window setting wrong;
- **High power**: not calling sleep;
- **Blurry**: temperature and humidity effects.


## 16. References

- Do not long-term DC bias (damages screen);

- Better display effect under direct sunlight (ambient light);
