# RS485


## 1. Module Overview

RS485 is a differential serial communication interface defined by the EIA/TIA-485 standard, used for multi-point communication in industrial environments. It is common in:
- Industrial automation (PLC, frequency inverters, servo drives);
- Instrumentation (electricity, water, gas meters);
- Building automation (BA);
- Sensor networks;
- Long-distance communication (< 1200 m);
- Noise-resistant environments.

Features:
- Differential signals (A / B lines);
- Half-duplex or full-duplex;
- Bus topology (multi-device);
- Long distance and high speed;
- Common-mode noise immunity;
- Industrial-grade noise immunity.


## 2. Working Principle and Use Cases

### 2.1 Physical Layer

- **A line**: inverting, the negative differential terminal;
- **B line**: non-inverting, the positive differential terminal;
- Differential voltage: |VA - VB| > 200 mV = logic 1; < -200 mV = logic 0;
- Common-mode voltage: -7V~+12V (the driver can withstand);
- Bus voltage: typically 0~5V.

### 2.2 Topology

- Bus topology (daisy chain), not star;
- 120Ω termination resistors (both ends of the bus);
- Transceiver control (DE / RE): direction switching is needed for half-duplex.

### 2.3 Speed vs Distance

| Speed | Distance |
|---|---|
| 10 Mbps | 15 m |
| 1 Mbps | 120 m |
| 100 kbps | 1200 m |
| 10 kbps | 12000 m |

## 5. Hardware Connection

### 5.1 Wiring

- Twisted pair for A and B (recommended);
- 120Ω termination resistors at both ends of the bus;
- Do not use a star topology (causes signal reflections);
- Avoid many stubs on short cables;
- Common ground (single-point grounding is usually recommended).

### 5.2 Termination Resistor

- One 120Ω resistor at each end of the bus (matched to cable characteristic impedance);
- Can be omitted for many stubs or short cables (impedance mismatch has little impact);
- 120Ω + 1 nF (suppresses high-frequency reflections).

### 5.3 Transceiver Control (DE / RE)

```text
Transmit: DE = high, RE = high, driver enabled
Receive:  DE = low,  RE = low,  receiver enabled
```

- Half-duplex: DE and RE are controlled together (most chips);
- Full-duplex: no DE / RE; TX and RX operate simultaneously;
- Auto-direction chips: hardware auto-switching, no MCU control needed.

### 5.4 Isolation

- Ground loop interference: common in long-distance / multi-device scenarios;
- Isolation solution: isolated DC-DC power + optocoupler / digital isolator (ISO3082, ADuM1411);
- Recommended: always add isolation for industrial applications;
- Note: both sides of the isolation need independent supplies (power isolation).

### 5.5 Protection

- TVS diodes (against surges);
- PTC self-resetting fuse;
- Common-mode choke (CMCC);
- ESD protection: ±15 kV (built into some industrial transceivers).


## 6. Communication Method

- Half-duplex (A / B shared);
- Full-duplex (A / B, Y / Z two pairs);
- Master / slave protocol (Modbus is common);
- Multi-master protocol (custom);
- Multi-point polling.


## 7. Initialization Flow

1. Configure GPIO (TX, RX, DE / RE);
2. Configure UART (baud rate, data bits, stop bits, parity);
3. Configure DE / RE as default-receive;
4. Test (send 0x55 and observe the waveform);
5. Start communication.


## 8. Driver Interface

```python
class RS485:
    def __init__(self, uart, de_pin):
        self.uart = uart
        self.de = de_pin

    def begin(self):
        self.de.low()  # default receive

    def write(self, data):
        """Send data."""
        self.de.high()  # transmit mode
        self.uart.write(data)
        self.uart.flush()
        self.de.low()  # switch back to receive

    def read(self, n=1):
        """Read data."""
        return self.uart.read(n)

    def any(self):
        return self.uart.any()


class ModbusRTU:
    """Modbus RTU master."""
    def __init__(self, rs485):
        self.rs485 = rs485
        self.timeout = 0.1

    def read_holding(self, slave, addr, qty):
        """Read holding registers (function code 0x03)."""
        frame = self._build_frame(slave, 0x03, addr, qty)
        self.rs485.write(frame)
        return self._parse_response()

    def write_register(self, slave, addr, value):
        """Write single register (function code 0x06)."""
        frame = self._build_frame(slave, 0x06, addr, value)
        self.rs485.write(frame)

    def _crc16(self, data):
        """Modbus CRC16."""
        crc = 0xFFFF
        for byte in data:
            crc ^= byte
            for _ in range(8):
                if crc & 1:
                    crc = (crc >> 1) ^ 0xA001
                else:
                    crc >>= 1
        return crc
```


## 9. Data Format and Units

- Byte stream (UART frame format);
- Protocol layer: Modbus RTU (see Modbus doc for details);
- Frame gap: 3.5 character times (Modbus RTU);
- Address: slave number (1~247);
- Function code + data + CRC16.


## 10. Interrupts and DMA

- Receive: interrupt + ring buffer;
- DMA: continuous reception (large data);
- Transmit: FIFO + switch to receive after completion;
- Bus idle detection: 3.5 character times with no data = end of frame.


## 11. Error Handling

- **CRC error**: retransmit;
- **No response**: wrong slave address, wrong speed, broken line;
- **Bus collision**: multiple masters transmitting simultaneously, arbitration;
- **Timeout**: master waits too long for response;
- **Noise interference**: add shielding, use isolation;
- **Missing termination resistor**: signal reflections cause bit errors.


## 12. Low Power

- Switch to receive mode when not communicating;
- Transceiver standby current < 1 mA;
- The whole system can power off the RS485 module.


## 13. Example Code

```python
rs485 = RS485(uart, de_pin)
rs485.begin()

# Send
rs485.write(b'\x01\x03\x00\x00\x00\x06\xC5\xC8')

# Receive
if rs485.any():
    data = rs485.read()
    print(data)
```


## 14. Debugging Methods

1. Measure A / B waveforms (oscilloscope differential probe);
2. Check the termination resistors (120Ω);
3. Check DE / RE switching timing;
4. Test the transceiver alone (loopback);
5. Connect to PC via USB-RS485 converter for debugging;
6. Check the isolated supply (for isolated versions).


## 15. Frequently Asked Questions

- **No data received**: DE / RE reversed, A / B reversed, termination resistor wrong;
- **Data error**: termination resistor wrong, distance too long, speed too high;
- **Some slaves unresponsive**: wrong address, wrong baud rate;
- **Multi-device conflict**: duplicate device addresses;
- **Bus unstable**: no isolation, no shielding;
- **Long-distance communication fails**: speed too high, add repeaters.


## 16. References

- Both ends of the bus must have termination resistors (with many stubs the design is non-trivial);
- Isolated supplies must be independent;
- Keep away from strong interference sources (frequency inverters, high-power motors);
- Bus speed is inversely proportional to distance.
