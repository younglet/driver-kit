# Communication Overview

Communication is the bridge that connects MCUs to peripherals, devices to devices, and endpoints to the cloud. DriverKit covers **12 communication modules**, grouped into **Bus Protocols** and **Protocols / Wireless** by form factor.

## Quick Index by Domain

### Bus (7)

Wired buses for chip-to-chip or on-board communication.

| Module | Description | Doc |
|---|---|---|
| GPIO | Most basic I/O and expansion | [Open](./通信-总线-GPIO) |
| I2C | Two-wire synchronous bus; mainstream for sensors | [Open](./通信-总线-I2C) |

## Selection Quick Reference

| Need | Recommended |
|---|---|
| Connect sensors | [I2C](./通信-总线-I2C) (multi-device) or [SPI](./通信-总线-SPI) (high-speed) |
| Debug serial port | [UART](./通信-总线-UART) + USB-TTL bridge |
| Multi-device on-board | [I2C](./通信-总线-I2C) (watch for address conflicts) |
| Industrial fieldbus | [RS485](./通信-总线-RS485) + [Modbus](./通信-协议-Modbus) |
| Automotive electronics | [CAN](./通信-总线-CAN) (preferred) |
| Video streaming | [Wi-Fi](./通信-无线-WiFi) or [USB](./通信-总线-USB) |
| Wearables | [Bluetooth BLE](./通信-无线-蓝牙BLE) |
| Remote monitoring | [LoRa](./通信-无线-LoRa) or NB-IoT |
| Positioning / time service | [GNSS](./通信-无线-GNSS) |
| PC comms / charging | [USB](./通信-总线-USB) |

## Common Knowledge

- **Level matching**: 3.3V MCU connecting to 5V devices needs level shifting
- **Termination resistors**: 120Ω at both ends of RS485/CAN bus
- **CRC / checksum**: All protocols include error detection
- **Bus arbitration**: CAN arbitrates by ID; I2C/UART are single-master
- **Antenna**: RF modules (BLE/Wi-Fi/LoRa/GNSS) require proper matching
