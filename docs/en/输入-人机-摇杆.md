# Joystick


## 1. Module Overview

A joystick is a device used for multi-axis input. Common use cases:

- Game controllers;
- Remote control devices (drones, robots);
- Industrial remote control (cranes, AGV);
- Medical equipment (CT, MRI operation);
- Simulated driving;
- Self-balancing scooter / unicycle control;
- Motion sensing control.

Types:

- **Dual-axis potentiometer joystick (PS2 Joystick)**: classic for makers;
- **Three-axis (with Z button)**: X/Y analog + Z digital button;
- **Hall joystick**: magnetic sensing, no mechanical wear;
- **Capacitive joystick**: high resolution;
- **Optical joystick**: high precision;
- **Force feedback joystick**: with torque feedback (high-end);
- **Force sensor joystick**: pressure sensing.


## 2. Working Principle and Use Cases

### 2.1 Potentiometer Joystick

- One potentiometer each for X axis and Y axis;
- Center position outputs mid-point voltage (e.g. 2.5V@5V);
- Movement changes resistance ratio;
- Output voltage is read via ADC.

### 2.2 Hall Joystick

- Magnet below the joystick;
- Magnetic-sensitive elements (Hall / AMR / TMR) detect magnetic field direction;
- Output proportional to position;
- No mechanical wear, long lifetime.

### 2.3 Z Axis Button

- Switch triggered when joystick is pressed down;
- SW pin typically low level indicates pressed.

## 5. Hardware Connection

### 5.1 Basic Circuit

- VRx: X axis analog output → MCU ADC;
- VRy: Y axis analog output → MCU ADC;
- SW: button → GPIO (input pull-up);
- VCC: 3.3V/5V;
- GND: ground.

### 5.2 Potentiometer Specification

- Most are 10 kΩ;
- Center voltage = VCC / 2;
- Min/max voltage = GND / VCC;
- Measurement range: 0~VCC.

### 5.3 Filtering

- Capacitor (100nF) parallel to VCC;
- Software median filter (remove spikes);
- Software moving average (noise reduction);
- Dead zone: center range directly outputs 0.


## 6. Communication Method

- Analog voltage: VRx, VRy (most common);
- I2C: Hall + dedicated IC;
- SPI: Hall + dedicated IC;
- PWM: a few;
- USB HID: game controllers.


## 7. Initialization Flow

1. Configure ADC (VRx, VRy);
2. Configure GPIO (SW input pull-up);
3. Read center position (calibration);
4. Set dead zone threshold;
5. Start reading.


## 8. Common Driver Interfaces

```python
class Joystick:
    def __init__(self, vrx_adc, vry_adc, sw_pin):
        self.vrx = vrx_adc
        self.vry = vry_adc
        self.sw = sw_pin
        self.center_x = 0
        self.center_y = 0
        self.deadzone = 50  # mV or ADC counts

    def begin(self):
        """Calibrate center."""
        # Sample center position
        sum_x = sum_y = 0
        for _ in range(100):
            sum_x += self.vrx.read()
            sum_y += self.vry.read()
        self.center_x = sum_x // 100
        self.center_y = sum_y // 100

    def read(self):
        """Returns (x, y) normalized to -100~+100."""
        raw_x = self.vrx.read()
        raw_y = self.vry.read()
        x = raw_x - self.center_x
        y = raw_y - self.center_y
        # Dead zone
        if abs(x) < self.deadzone:
            x = 0
        if abs(y) < self.deadzone:
            y = 0
        # Normalize
        x_norm = x * 100 / (self.center_x)  # Assume symmetric range
        y_norm = y * 100 / (self.center_y)
        return x_norm, y_norm

    def is_pressed(self):
        return self.sw.value() == 0
```

## 9. Data Format, Units, and Timing

- Analog: voltage (mV / ADC counts);
- Digital: normalized (-100~+100);
- Angle: ° (±30° typical);
- State: pressed / not pressed.


## 10. Calibration, Compensation, and Filtering

- **Center calibration**: sample N times and average at startup;
- **Range calibration**: record max / min values → normalize;
- **Dead zone**: eliminate center jitter (center ±5% no response);
- **Temperature compensation**: potentiometer resistance affected by temperature;
- Save calibration data to Flash.



### 11.1 Median Filter

```python
def median_filter(samples):
    return statistics.median(samples)
```

### 11.2 Moving Average

```python
def moving_avg(buf, new):
    buf.append(new)
    if len(buf) > N:
        buf.pop(0)
    return sum(buf) / len(buf)
```

### 11.3 Dead Zone + Hysteresis

```python
def deadzone(value, threshold):
    if abs(value) < threshold:
        return 0
    return value

def hysteresis(value, last, threshold):
    if abs(value - last) < threshold:
        return last
    return value
```


## 11. Error Handling

- **Jitter**: filtering;
- **Center drift**: temperature, mechanical wear, recalibrate;
- **Button bounce**: debounce;
- **Maximum voltage insufficient**: supply voltage low or potentiometer aged;
- **Output stuck**: mechanical wear, potentiometer failure.


## 12. Low Power

- No need for continuous sampling, can be timed;
- Analog joystick very low power (µA level);
- Hall joystick mA level (during measurement).

## 14. Debugging Methods

1. Use multimeter to measure VRx, VRy voltage (center 2.5V@5V);
2. Use oscilloscope to view waveform;
3. Measure ADC counts (0~4095);
4. Test max travel voltage;
5. Test button on / off;
6. Test filter effect.


## 15. Frequently Asked Questions

- **Center offset**: potentiometer aged, recalibrate;
- **Jitter**: insufficient filtering, add hardware filter capacitor;
- **Reversed direction**: swap X/Y;
- **Button no response**: pull-up not connected, GPIO configuration wrong;
- **Potentiometer failure**: mechanical wear, needs replacement;
- **Insufficient travel range**: potentiometer resistance mismatch.


## 16. References

- Potentiometer has limited lifetime (10⁴~10⁶ cycles);
- Center may drift after long-term use;
- Recommend Hall joystick for high-frequency vibration environments;
- Operating force reflects feel, choose different operating forces for different applications;
- Choose industrial grade for dust / waterproof requirements.
