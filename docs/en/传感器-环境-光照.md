# Ambient Light Sensor


## 1. Module Overview

An ambient light sensor measures the intensity of ambient light (in lux) or the visible + IR component. It is commonly used for:
- Automatic brightness adjustment (phones, tablets, laptops)
- Automatic street / corridor light switching
- Camera white-balance / exposure assist
- Greenhouse light monitoring
- Display backlight dimming
- Solar tracking

Types:
- **Photoresistor (LDR)**: CdS element whose resistance varies with light intensity
- **Photodiode**: current proportional to light intensity
- **Phototransistor**: amplified current output
- **Integrated IC**: BH1750, TSL2561, TSL2591, MAX44009, VEML7700, etc. — built-in ADC + IR rejection


## 2. Working Principle and Use Cases

- **LDR**: CdS material, dark resistance ~1 MΩ, bright resistance ~1 kΩ. Requires ADC sampling.
- **Photodiode**: under reverse bias the current varies with light intensity; transimpedance amplifier output.
- **Integrated IC**: silicon photodiode array + ADC + I2C lux output with built-in IR rejection; high accuracy.

Applications:
- Auto dimming (phone backlight, vehicle displays)
- Automatic street / corridor lighting
- Smart curtains / blinds
- Smart home (environment awareness)
- Greenhouse photosynthetically active radiation (PAR) monitoring
- Industrial assembly-line product identification

| Resolution | 16/20/24 bit |

| IR rejection | Affects accuracy under indoor and artificial light |
| Output data rate | 1 Hz to several hundred Hz |
| Interface | I2C (most), analog |
| Power | 1.8V / 3.3V |
| Power consumption | Measurement µA, standby nA |
| Field of view | 50°~120° |
| Spectral response | Close to human-eye 555 nm sensitivity (V-lambda filtering) |

**Selection suggestions**:
- Entry / simple: photoresistor + ADC
- Indoor auto dimming: BH1750, VEML7700, OPT3001
- High dynamic range: TSL2591, VEML6035, OPT4048
- Color identification: TCS34725, APDS-9960
- Spectral: AS7341
- Ultra-low power: MAX44009, OPT3002


## 5. Hardware Connection

- I2C pull-ups: 4.7kΩ
- ADDR pin: most chips allow toggling the I2C LSB address
- INT: threshold interrupt output
- Optical window: keep clear of obstructions
- Dark current: photoresistor needs a ~100 kΩ range voltage-divider resistor
- IR filter: VEML7700 etc. have built-in IR rejection
- Directional sensitivity is significant (side-mount vs top-mount differs greatly)


## 6. Communication Method

- Analog output: voltage signal, sampled by ADC
- I2C: standard 100 / 400 kHz
- Some chips support SMBus Alert interrupt
- Integrated IC outputs lux or raw counts


## 7. Initialization Flow

1. Configure I2C
2. Read WHO_AM_I
3. Soft-reset
4. Set measurement mode (single / continuous)
5. Set integration time / gain
6. Start measurement
7. Read lux or raw value

## 9. Data Format and Units

- Illuminance unit: lux
- 1 lux = 1 lm/m²
- Outdoor sunlight: 10,000~100,000 lux
- Indoor lighting: 100~500 lux
- Moonlight: 0.1~1 lux
- Different light sources: incandescent light is close to the 555 nm human-eye peak; LEDs have a strong blue bias


## 10. Calibration and Filtering

- Integrated ICs are factory-calibrated
- High-precision needs: calibrate against a known illuminance source (lux meter)
- Note: phone auto-brightness uses local algorithms that may differ from raw sensor readings
- Temperature effect: small


## 11. Interrupts and DMA

- Threshold interrupt: triggers INT when low / high threshold is crossed
- Some chips support persistence (interrupt only after N consecutive crossings)
- Auto dimming: set different tier thresholds


## 12. Error Handling

- I2C error: check address and pull-ups
- Data saturation: outdoor strong light
- Reading 0: total darkness or sensor failure
- Drift: temperature, aging


## 13. Low Power

- Single measurement followed by sleep
- Standby < 1 µA
- 1 sample / second is enough for most scenarios

## 15. Debugging Methods

1. Indoor with vs without shading and light
2. Cover with hand and observe lux decrease
3. Read in sunlight (watch the range)
4. Cross-check with a phone lux-meter app
5. Shine a flashlight and observe transient response


## 16. Frequently Asked Questions

- **BH1750 reading unstable**: integration time not configured
- **Outdoor saturation**: range insufficient; add a neutral-density filter
- **TSL2561 reading too high**: IR not rejected; use the dual-channel method
- **I2C error**: wrong ADDR pin level
- **Photoresistor responds slowly**: physical characteristic, hundreds of milliseconds

## 18. Algorithms

### 11.1 V-lambda Correction

The human eye is most sensitive at 555 nm (green); the sensor output should approximate the V(λ) curve.

### 11.2 IR Compensation

Typical light sources contain IR; indoor auto-dimming requires IR rejection or compensation (the TSL2561 dual-channel method).

### 11.3 Dual-channel Method

VIS + IR channels → estimate visible light → reject IR interference.
