# Servo


## 1. Module Overview

A servo is a position servo integrating a DC motor + gearbox + control circuit + position feedback, controlled via PWM. Common uses:

- Robot joints
- RC models (cars, boats, planes)
- Servo robotic arms
- Smart car steering
- Camera gimbals
- DIY toys

Types:

- **Analog servo**: 50 Hz PWM control
- **Digital servo**: MCU-built-in PID, fast and precise
- **Plastic-gear servo**: low cost
- **Metal-gear servo**: high torque, long life
- **Bus servo**: serial communication, multi-servo daisy-chain (e.g., LewanSoul/Feetech SCS, AX, LX series)


## 2. Working Principle and Use Cases

Internal structure: DC motor + reduction gear train + position potentiometer (feedback) + control circuit board.

PWM input:

- 50 Hz (20 ms period)
- Pulse width 0.5~2.5 ms (some servos 1.0~2.0 ms)
- Center 1.5 ms = 90° (or servo center position)
- 0.5 ms = 0°, 2.5 ms = 180°
- Range adjustable (e.g., 270°, 360°)

The control circuit compares the PWM pulse width with the potentiometer position and drives the motor to the target position.

## 5. Hardware Connection

- Three-wire interface: signal (orange/yellow), VCC (red), GND (brown/black)
- Independent power: servos draw large current; an independent supply (5V/6V, 1 A+) is recommended
- Common ground with MCU: required
- Signal level: typically 3.3V~5V compatible
- PWM frequency: 50 Hz (some support 200 Hz~300 Hz high-speed servos)
- Flyback capacitor: large bulk capacitor on supply (e.g., 470µF)
- Prevent stall burn-out: observe torque limits
- Current limiting: fuse or software stall protection


## 6. Communication Method

### 6.1 PWM (analog / digital servo)

50 Hz, single pulse width controls angle.

### 6.2 Bus Servo (serial)

- One data line (half-duplex)
- Multi-servo daisy chain
- Protocols: LewanSoul LSC, Feetech SCS, AX, LX, Dynamixel, etc.
- Read/write position, speed, torque, temperature, ID, baud rate


## 7. Initialization Flow

### 7.1 PWM Servo

1. Configure PWM (50 Hz, 0.5~2.5 ms pulse)
2. Output center (1.5 ms)
3. Vary signal to control angle

### 7.2 Bus Servo

1. Configure UART (9600 / 115200)
2. Set servo ID (default 1 from factory)
3. Set baud rate (some are adjustable)
4. Set motion mode (position / speed / torque)
5. Write position command
6. Read status


## 8. Driver Interface

```python
class Servo:
    def __init__(self, pwm):
        self.pwm = pwm

    def begin(self):
        self.write(90)  # center

    def write(self, angle):
        """angle 0~180."""
        us = int(500 + (angle / 180) * 2000)  # 0.5~2.5 ms
        self.pwm.duty_us(us)

    def write_us(self, us):
```

## 9. Data Format, Units, and Timing

### 9.1 PWM

Pulse width µs: 500~2500 µs corresponds to 0~180°.

### 9.2 Bus Servo

LewanSoul LSC protocol (Feetech similar):

- Frame: 0x55 0x55 [length] [command] [params...n] [checksum]
- Position: 0~1000 (corresponds to 0~240°)
- Speed: 0~1000 + direction
- Temperature: °C
- Voltage: ×10 integer

Dynamixel protocol (more complex):

- Multi-instruction read/write
- Indirect addressing
- Feedback: position / speed / torque / temperature / voltage, etc.


## 10. Error Handling and Exception Recovery

- Jitter: unstable signal, supply ripple, pulse glitches
- No response: wrong signal frequency, pulse out of range
- Jitter near center: dead-band adjustment
- Bus servo communication error: check ID, baud rate, wiring, termination resistor
- Stall burn-out: torque protection, limit switches, stall detection


## 11. Low Power

- Bus servos support torque-off
- PWM servos consume power holding position when idle
- Lower working voltage to save energy (with reduced torque)

## 13. Debugging Methods

1. Test 0° / 90° / 180° positions
2. Check torque (by hand)
3. Test response speed
4. Check signal waveform (oscilloscope)
5. Bus servo: use serial monitor to view commands


## 14. Frequently Asked Questions

- **Jitter**: unstable signal, add supply filtering
- **No response**: PWM frequency wrong, pulse out of range
- **Weak**: supply voltage insufficient, current limit
- **Overload burn-out**: prolonged stall
- **Bus servo communication failure**: ID conflict, baud rate mismatch, wiring wrong
- **Inaccurate centering**: potentiometer aged or center offset
- **Intermittent position loss**: insufficient supply current
