# Buzzer


## 1. Module Overview

A buzzer is a sound-producing device used for alarms, prompts, and music playback. Common uses:

- Home-appliance prompt tones
- Doorbells
- Alarms
- Countdown timers
- Electronic toys
- Instrumentation
- Smart home
- Simple music playback

Types:

- **Active buzzer**: built-in oscillator, sounds when powered (DC-driven)
- **Passive buzzer**: requires external PWM drive
- **Electromagnetic**: most common
- **Piezo**: thin slice vibrating, high frequency
- **Piezo speaker**: continuous voice / music playback


## 2. Working Principle and Use Cases

### 2.1 Electromagnetic Buzzer

Current through a coil produces a magnetic field that drives a diaphragm to vibrate and produce sound.

- Active: contains internal oscillator (typically 2~4 kHz)
- Passive: needs external PWM signal

### 2.2 Piezo Buzzer

Piezo material (ceramic) deforms when voltage is applied. A high-frequency square wave produces sound.

## 5. Hardware Connection

### 5.1 Active Buzzer

- Direct GPIO drive (high level sounds)
- Watch the current (5V active ~30 mA; some GPIOs can't source enough — add a transistor)
- Modular versions usually include a transistor; direct IO is enough
- Reverse-parallel diode (usually built-in)

### 5.2 Passive Buzzer

- Requires external PWM drive
- 50% duty cycle
- Frequency determines pitch
- Also watch the drive current

### 5.3 Piezo Buzzer

- Direct IO drive (some need level shifting)
- High-frequency square wave
- Small current

### 5.4 Speaker

- Needs an amplifier (e.g., PAM8403)
- Cannot connect directly to MCU
- 8Ω speaker paired with 8Ω amplifier output


## 6. Communication Method

- **GPIO**: active buzzer on/off
- **PWM**: passive buzzer / piezo tone control
- **I2S**: digital audio (advanced playback)
- **DAC + amplifier**: analog audio
- **MP3 module**: pre-stored audio (e.g., DFPlayer Mini)


## 7. Initialization Flow

1. Configure GPIO (output mode) / PWM channel
2. Set initial state to off
3. Apply signal to make sound
4. Turn off to stop


## 8. Driver Interface

```python
class Buzzer:
    """Active buzzer."""
    def __init__(self, pin, active_high=True):
        self.pin = pin
        self.active_high = active_high

    def begin(self):
        self.off()

    def on(self):
        if self.active_high:
            self.pin.high()
        else:
            self.pin.low()

    def off(self):
        if self.active_high:
            self.pin.low()
        else:
            self.pin.high()


class Tone:
    """Passive buzzer (PWM driven)."""
    def __init__(self, pwm):
        self.pwm = pwm

    def begin(self):
        pass

    def tone(self, freq_hz, duration_ms=0):
        """Play a given frequency."""
        period_us = 1_000_000 // freq_hz
        self.pwm.freq(freq_hz)
        self.pwm.duty(50)  # 50% duty cycle
        if duration_ms > 0:
            time.sleep_ms(duration_ms)
            self.pwm.duty(0)

    def no_tone(self):
        self.pwm.duty(0)


class Melody:
    """Simple melody player."""
    NOTES = {
        'C4': 262, 'D4': 294, 'E4': 330, 'F4': 349,
        'G4': 392, 'A4': 440, 'B4': 494,
        'C5': 523, 'D5': 587, 'E5': 659, 'F5': 698,
        'G5': 784, 'A5': 880, 'B5': 988,
        'REST': 0,
    }

    def __init__(self, tone):
        self.tone = tone

    def play_note(self, note, duration_ms):
        if note == 'REST':
            self.tone.no_tone()
        else:
            self.tone.tone(self.NOTES[note], duration_ms)
        time.sleep_ms(int(duration_ms * 0.3))  # gap
```

## 10. Error Handling and Exception Recovery

- **No sound**: power not connected, no PWM output, signal wrong
- **Quiet**: insufficient voltage, insufficient drive current
- **Poor sound quality**: frequency inaccurate, PWM duty cycle not 50%
- **Noise**: power supply ripple, unstable signal


## 11. Low Power

- Power off when silent
- Intermittent sound saves energy
- Volume adjustment (PWM duty cycle affects volume)

## 13. Debugging Methods

1. Measure drive voltage / signal
2. Listen for the correct frequency
3. Measure volume (phone decibel meter)
4. Check current (sufficient to drive)
5. Check whether resonant frequency matches


## 14. Frequently Asked Questions

- **No sound**: power wrong, no signal output
- **Quiet**: insufficient drive current
- **Poor sound quality**: PWM duty cycle not 50%, frequency inaccurate
- **Noise**: power supply ripple, signal jitter
- **Speaker silent**: amplifier not connected, wiring wrong
