# Barometric Pressure Sensor


## 1. Module Overview

A barometric pressure sensor measures ambient atmospheric pressure. It is commonly used in altimeters, weather stations, drone altitude hold, GPS augmentation, indoor navigation, outdoor / mountaineering devices, and phone barometers.

Standard sea-level pressure is about 1013.25 hPa; pressure drops roughly 1 hPa every 8 m of altitude gain (the relationship between pressure and altitude is non-linear).


## 2. Working Principle and Use Cases

- **MEMS piezoresistive**: a silicon diaphragm deforms under pressure, changing its resistance
- **MEMS capacitive**: diaphragm gap changes with pressure, changing the capacitance (low power, high precision)
- **Piezoelectric**: change in acoustic resonance frequency (high-precision industrial)

Applications:
- Drone / projectile altitude hold
- Outdoor-watch altimeter
- Indoor floor detection (map matching)
- Weather station
- Vacuum / sealed-device leak detection
- Medical ventilator

| Power | 1.8V~3.6V / 3V~5V |

**Selection suggestions**:
- Entry level: BMP180, BMP280
- High precision: MS5611, BMP388, BMP390, ICP-10111
- Three-in-one: BME280
- Waterproof: ICP-10111
- Low power: DPS310, LPS22HB


## 5. Hardware Connection

- I2C pull-ups: 4.7kΩ
- BMP280 / BME280 default address 0x76 (SDO=GND) or 0x77 (SDO=VDD)
- Avoid vibration interference: MEMS is sensitive to vibration
- Module openings: allow the sensor to sense ambient pressure
- Waterproof versions have a PTFE membrane — plan the opening layout
- Stay away from DC-DC converters, motors, and other pressure sources


## 6. Communication Method

- I2C: standard / fast mode
- SPI: 4-wire, mode 0 or 3
- Data package: raw pressure + raw temperature + calibration coefficients


## 7. Initialization Flow

1. Read WHO_AM_I (0x58 / 0x60)
2. Soft-reset
3. Read calibration coefficients (stored in NVM at the factory)
4. Set operating mode (sleep / forced / continuous)
5. Set pressure and temperature oversampling rate (OSR)
6. Set output data rate
7. Set filter
8. Trigger measurement (e.g. forced mode)
9. Read raw data and run compensation

### 10.1 International Standard Atmosphere (ISA) Model

```text
P = P0 * (1 - L*h/T0)^(g*M/(R*L))
```

Simplified formula:

```python
import math

def altitude_from_pressure(p, p0=1013.25):
    """Simplified formula, accuracy ±10 m."""
    return 44330.0 * (1 - (p / p0) ** (1 / 5.255))
```

### 10.2 Improving Accuracy

Use the local real-time pressure (from a weather station or GPS) as P0 rather than the standard value.

### 10.3 Temperature Compensation

For high-precision needs, account for the temperature effect on air density and use the full Barometric Formula.


## 9. Data Format and Units

- Unit: Pa (Pascal) or hPa (hectopascal)
- 1 hPa = 100 Pa
- Standard sea-level pressure: 1013.25 hPa
- Temperature: °C
- Data flow: raw values + calibration coefficients → compensation → actual values


## 10. Calibration and Filtering

- Factory calibration coefficients are stored in NVM; no user calibration required
- High-precision needs: calibrate against a known pressure reference (e.g. weather station data)
- Sea-level reference: use the local weather station data
- Long-term drift: typically < 1 hPa per year — negligible


## 11. Interrupts and DMA

- Some chips support a data-ready interrupt
- FIFO: a few chips support it (MS5611 has no FIFO but reads fast)
- Single-shot measurement: forced mode is recommended


## 12. Error Handling

- WHO_AM_I error: wrong I2C address, missing pull-ups
- Data overflow: out of range (e.g. low-pressure environment < 300 hPa)
- Reading jumps: vibration interference, blocked opening
- Altitude drift: temperature and humidity variations


## 13. Low Power

- Sleep mode < 1 µA
- Forced mode: trigger on demand
- Intermittent measurement: 1 Hz is enough for outdoor altimeters

## 15. Debugging Methods

1. Read WHO_AM_I to verify
2. Compare indoor readings with weather station data
3. Go up/down stairs and observe altitude changes
4. Heat / cool the sensor and observe temperature response
5. Long-term monitor the bias stability


## 16. Frequently Asked Questions

- **Altitude drifts**: temperature affects pressure; apply temperature compensation or use a local P0
- **Data jumps**: opening blocked (e.g. dust mesh clogged)
- **BMP280 / BME280 confusion**: BME280 is 0x60, BMP280 is 0x58
- **Large altitude error**: not using a local pressure reference
- **Condensation damage**: moisture on the diaphragm at high humidity — use a waterproof model
