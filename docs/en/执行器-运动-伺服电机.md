# Servo Motor


## 1. Module Overview

A servo motor is a high-precision closed-loop motor system integrating motor + encoder + driver. Common uses:

- Industrial robot joints
- CNC machine tool spindle / feed
- Printing and packaging machinery
- Semiconductor equipment
- Automated production lines
- AGV / AMR drive
- Medical equipment
- High-end CNC / laser equipment

Types:

- **AC servo**: mainstream, industrial grade
- **DC servo**: early generation, less common now
- **Permanent Magnet Synchronous Servo (PMSM)**: mainstream
- **Asynchronous servo**: low-cost scenarios
- **Stepper servo**: stepper + encoder (low-cost high-precision)
- **Linear servo**: linear motion


## 2. Working Principle and Use Cases

A servo system consists of a servo driver + servo motor:

- Motor (permanent-magnet synchronous PMSM or asynchronous)
- High-precision encoder (17~23 bit, 131072~8388608 pulses per revolution)
- Driver with built-in PID / FOC control
- Accepts position / speed / torque commands and outputs closed-loop control

Control modes:

- **Position mode**: precise angle / distance control
- **Speed mode**: constant rotation speed
- **Torque mode**: constant torque output
- **Hybrid mode**: position + torque, speed + torque, etc.


## 3. Key Parameters and Selection Guide

| Parameter | Description |
|---|---|
| Rated power | 50W ~ 22kW |
| Rated torque | N·m |
| Rated speed | rpm (typically 1500 / 3000) |
| Max speed | rpm |
| Encoder | 17~23 bit incremental / absolute |
| Inertia class | low / medium / high inertia |
| Brake | yes / no |
| Control mode | position / speed / torque |
| Communication | pulse + direction / RS485 / CANopen / EtherCAT / MECHATROLINK |
| Feedback precision | ±1 pulse (high-resolution encoder) |
| Speed response frequency | 1 kHz+ |
| Working temperature | 0~40°C (driver) / 0~55°C (motor) |

**Selection suggestions**:

- Beginner / small machinery: Delta ASDA-B2, Inovance IS620N, stepper servo
- Medium machinery: Inovance SV660, Delta ASDA-A2, Yaskawa Σ-7
- High-end / CNC: Mitsubishi J4, Yaskawa Σ-X, Siemens S120
- Bus control: EtherCAT, CANopen
- Pulse control: low-cost solution


## 4. Hardware Connection

- Three-phase power: 220V AC single-phase / 380V AC three-phase (per driver spec)
- Control power: DC 24V (some drivers need it)
- Encoder cable: twisted-pair shielded, impedance-matched
- Motor power cable: thick (e.g., 1.5~6 mm²), short distance
- Grounding: PE grounding required
- Brake resistor: external when braking energy is high
- Anti-interference: shielded encoder cable, run separately from power cable


## 5. Communication Method

### 6.1 Pulse + Direction (traditional)

- PUL (pulse) + DIR (direction)
- Differential (RS422): noise-immune
- Single-ended: low cost
- Pulse frequency: 500 kHz ~ 4 MHz

### 6.2 Bus Communication

- **Modbus RTU**: universal protocol
- **CANopen**: mainstream in Europe
- **EtherCAT**: high-speed real-time (µs level)
- **MECHATROLINK**: mainstream in Japan
- **PROFINET / PROFIBUS**: Siemens systems
- **RS485 / RS422**: low-cost bus

### 6.3 Analog

±10V speed / torque reference


## 6. Initialization Flow

1. Wiring (power, motor, encoder, control signals)
2. Power on
3. Set motor parameters (model, encoder pole pairs)
4. Set control mode (position / speed / torque)
5. Set PID parameters (auto-tune or manual)
6. Set input signals (pulse + direction / bus)
7. Set electronic gear ratio
8. Save parameters
9. Test run

Set the command pulses corresponding to one motor revolution:

```text
Motor 1 revolution = encoder P/R × 4 quadrature = 131072 × 4 = 524288 (17-bit)
Electronic gear ratio = 524288 / desired pulses per revolution
```

For example, 10,000 pulses per revolution: 524288 / 10000 ≈ 52.4.



- Auto-tuning: many drivers have an "auto" button (one-click tuning)
- Manual tuning: tune speed-loop PI first, then position-loop P
- Rigidity level: low (soft) ~ high (rigid)
- Inertia ratio: load / motor inertia ratio; < 30 is easier to tune


## 8. Data Format, Units, and Timing

- Pulse: square wave (pulse + direction)
- Modbus RTU: register read/write (function codes 0x06 / 0x03 / 0x10)
- Position unit: pulses (convert by encoder bit count)
- Speed: rpm
- Torque: % or N·m


## 9. Error Handling and Exception Recovery

- **Over-current**: mechanical jam or motor short
- **Overload**: long-time over-rated torque
- **Encoder error**: encoder broken, noise
- **Under-voltage / over-voltage**: power abnormality
- **Overspeed**: above max speed
- **Lost steps** (stepper servo): excessive load
- Alarm codes: see driver manual
- Reset: clear alarm + re-enable



- Servo systems have high energy; emergency stop (E-stop) is mandatory
- Encoder wire break can cause uncontrollable motion; monitor in software
- Trial run with no load first, then light load, then heavy load
- Prevent over-travel (mechanical limit)


## 10. Low Power

- Disable when idle
- Energy recovery during braking (some drivers)
- Driver no-load loss

## 12. Debugging Methods

1. Power on, confirm no alarm
2. JOG mode test
3. Auto-tune (PID)
4. Load test (light ~ heavy)
5. Following error (position deviation)
6. Long-time temperature check
7. Encoder waveform test


## 13. Frequently Asked Questions

- **Motor doesn't turn**: not enabled, alarm not cleared, control signal wrong
- **Large following error**: PID misconfigured, electronic gear ratio wrong, insufficient rigidity
- **Vibration / noise**: PID parameters wrong, inertia ratio too large
- **Overload alarm**: mechanical jam, excessive load
- **Encoder alarm**: encoder cable fault, interference
- **Bus communication failure**: wrong address, baud rate mismatch, missing termination resistor
- **Runaway**: control signal wired wrong, servo parameter wrong
