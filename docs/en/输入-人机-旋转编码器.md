# Rotary Encoder


## 1. Module Overview

A rotary encoder is an input device that measures rotational angle or angular velocity. Common use cases:

- Volume knob;
- Industrial control knob;
- CNC machine handwheel;
- Motor speed measurement;
- Robot joint;
- Radar antenna;
- Camera focus ring;
- Self-balancing scooter / unicycle (pedal position).

Types:

- **Incremental encoder**: AB phase output, calculates angle and direction based on pulse count;
- **Absolute encoder**: each position has a unique code (see "Encoder" document);
- **Mechanical rotary encoder**: knob type, EC11/EC12/EC16 etc.;
- **Optical**: high resolution;
- **Magnetic**: contamination-resistant;
- **Capacitive**: high precision contactless.


## 2. Working Principle and Use Cases

### 2.1 Incremental

- Two switches A and B with specific phase offset;
- When rotating, A and B output square waves with 90° phase difference (quadrature);
- Clockwise: A leads B;
- Counter-clockwise: B leads A;
- Counting: check B level on A rising edge → direction;
- 4x decoding: count both rising and falling edges of A and B (resolution ×4).

### 2.2 CPR / PPR

- **CPR**: cycles per revolution;
- **PPR**: pulses per revolution;
- After 4x decoding: resolution = CPR × 4;
- E.g. EC11 = 20 CPR, 4x decoding = 80 pulses/revolution.

### 2.3 Speed Measurement

- **M-method (frequency measurement)**: count pulses within fixed time window, suitable for high speed;
- **T-method (period measurement)**: measure interval between adjacent pulses, suitable for low speed;
- **M/T method**: combine both, high precision.

## 5. Hardware Connection

### 5.1 Basic Circuit

- A, B two signal lines connected to MCU;
- Internal pull-up resistors (most EC11/EC12);
- Switch (SW): optional;
- Common terminal COM: connect to GND or VCC (depending on circuit);
- Series resistor: 100Ω current limiting;
- TVS for ESD protection.

### 5.2 Long-distance Transmission

- Add Schmitt trigger (74HC14);
- Differential transmission (high speed / long distance);
- Twisted pair to reduce interference.

### 5.3 Anti-interference

- Filter capacitor (100nF);
- Shielded wire;
- Software filtering.


## 6. Communication Method

- AB phase (direct GPIO);
- I2C: AS5600 etc. magnetic encoders;
- SPI: AS5048A etc.;
- SSI: absolute encoder;
- Analog: a few simple encoders.


## 7. Initialization Flow

1. Configure GPIO (A, B input + pull-up/pull-down);
2. Configure interrupt (A/B edge trigger);
3. Counter reset to zero;
4. Start.


## 8. Common Driver Interfaces

```python
class RotaryEncoder:
    """Incremental rotary encoder."""
    def __init__(self, pin_a, pin_b, pin_sw=None):
        self.a = pin_a
        self.b = pin_b
        self.sw = pin_sw
        self.count = 0
        self.last_a = False
        self.last_dir = 0  # -1, 0, +1

    def begin(self):
        """Initialize GPIO and interrupts."""
        pass

    def update(self):
        """Called by interrupt or timer."""
        a = self.a.value()
        b = self.b.value()
        # State machine decoding
        if a != self.last_a:
            if a == b:
                self.count += 1
                self.last_dir = 1
            else:
                self.count -= 1
                self.last_dir = -1
        self.last_a = a

    def read_count(self):
        return self.count

    def reset(self):
        self.count = 0

    def is_pressed(self):
        return self.sw and self.sw.value() == 0


class SpeedEncoder:
    """Speed measurement encoder (M method)."""
    def __init__(self, encoder, cpr=20):
        self.encoder = encoder
        self.cpr = cpr

    def speed_rpm(self, dt_ms):
        """Returns rpm."""
        delta = self.encoder.read_count()
        return (delta * 60 * 1000) / (self.cpr * dt_ms)
```



### 12.1 M Method (Frequency)

```python
def m_method(count_delta, dt_seconds, cpr):
    """Pulses within a fixed time window."""
    return count_delta / (dt_seconds * cpr) * 60  # rpm
```

### 12.2 T Method (Period)

Measure time between adjacent pulses:

```python
def t_method(pulse_period, cpr):
    """Time per pulse."""
    return 60 / (pulse_period * cpr)  # rpm
```

### 12.3 M/T Method

M method + T method combined, wide speed range.


## 9. Data Format, Units, and Timing

- Incremental: pulse count (signed to indicate direction);
- Speed: pulse count / time = frequency → rpm;
- Angle: pulse count / CPR × 360°.


## 10. Calibration, Compensation, and Filtering

- Median filter: remove pulse noise;
- Moving average: reduce speed jitter;
- Kalman filter: state estimation (angle + velocity).


## 11. Interrupts, DMA, FIFO, and Buffers

- Edge interrupt: A/B any edge trigger (4x decoding);
- Timer interrupt: periodic speed calculation;
- DMA: high-speed counting.

## 12. Error Handling

- **Bouncing**: state machine filter (required);
- **Mis-counting**: long-line interference, add filtering;
- **Missing counts**: interrupt handling too long;
- **A/B reversed**: direction reversed, swap in software or hardware;
- **Switch bouncing**: debounce.


## 13. Low Power

- Turn off unused timers;
- Button interrupt wake-up (encoder can serve as wake source);
- Enter sleep after long inactivity.

## 15. Debugging Methods

1. Manually rotate to test count change;
2. Clockwise +1, counter-clockwise -1;
3. Use oscilloscope to view A, B waveform;
4. Verify 4x decoding (counts × 4 per revolution = 80);
5. Test button press / release.


## 16. Frequently Asked Questions

- **Reversed direction**: A, B wiring reversed or software reverse;
- **No counting**: GPIO configuration wrong, interrupt not enabled;
- **Count jumps**: bouncing not eliminated;
- **Button no response**: button wiring wrong (most have button), GPIO mode wrong;
- **Missing counts at high speed**: interrupt handling too slow or MCU frequency insufficient.


## 17. References
