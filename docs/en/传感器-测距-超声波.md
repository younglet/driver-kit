# Ultrasonic Distance Sensor


## 1. Module Overview

An ultrasonic distance sensor emits an ultrasonic pulse, receives the reflection, and computes distance from the time-of-flight. Common uses:
- Robot obstacle avoidance
- Reverse parking radar
- Liquid-level measurement
- Distance alarm
- Smart car
- Industrial ranging

Advantages: unaffected by color or light. Drawbacks: wide beam cone, hard to detect small targets, severe absorption by foam / fabric.


## 2. Working Principle and Use Cases

The speed of sound in air is about 343 m/s (at 20°C). The distance formula:

```text
distance = (echo time × sound speed) / 2
```

Since air temperature affects the speed of sound, precise ranging requires temperature compensation:

```text
sound speed = 331.3 + 0.606 × T (m/s)
T in degrees Celsius
```

## 5. Hardware Connection

- HC-SR04: VCC (5V), GND, Trig (input), Echo (output)
- Note: HC-SR04's Echo is 5V; connect to a 3.3V MCU via level shifting (resistor divider or BSS138 MOSFET)
- JSN-SR04T: 3.3V / 5V compatible
- Waterproof probe comes with a separate cable (5 m / 10 m options)
- Keep away from noise sources (motors, fans, buzzers)
- Hardware / software filtering: remove glitches (take the median of multiple measurements)


## 6. Communication Method

- Trig/Echo: trigger with a 10 µs high pulse; Echo emits a high pulse whose width is proportional to distance
- UART: 9600~115200 baud, send ASCII command (e.g. 0x55), receive the distance
- I2C: MB series, KS103, etc.


## 7. Initialization Flow

1. Configure GPIO (Trig output, Echo input)
2. Configure the timer to capture the Echo pulse width
3. Configure UART / I2C (if used)
4. Start periodic measurement
5. Temperature compensation (optional)

## 9. Data Format and Units

- Unit: cm or mm
- HC-SR04 Echo high pulse width = distance (cm) × 58 µs (round trip, cm/µs)
  - In practice: distance (cm) = time (µs) / 58
- MB series UART outputs ASCII strings (e.g. "R123\r\n")
- I2C outputs a 16-bit value (cm or mm)


## 10. Calibration and Filtering

### 10.1 Median Filter

```python
import statistics

def median_filter(samples):
    return statistics.median(samples)
```

### 10.2 Moving Average

```python
def moving_avg(buf, new):
```

### 10.3 Kalman Filter

A 1D Kalman filter can be used in complex scenarios to remove pulse noise.

```python
def sound_speed(t_celsius):
    return 331.3 + 0.606 * t_celsius  # m/s

def distance_us(time_us, t=20):
    c = sound_speed(t) * 1e-6 * 1e2  # cm/µs
    return time_us * c / 2
```


## 11. Interrupts and DMA

- Use a timer to capture the Echo rising and falling edges
- Single measurement takes about 30 ms (for 5 m range)
- Measurement interval should be > 60 ms to avoid echo overlap


## 12. Error Handling

- Echo timeout (> 30 ms): target too far or absorption
- Distance too small (< 2 cm): blind zone
- Distance jumps: target edge or multiple reflection paths
- Measurement failure: sensor not connected or unstable power


## 13. Low Power

- Power on only when measuring
- Idle current < 2 mA
- Use a MOSFET to switch VCC


## 14. Example Code

```python
class Ultrasonic:
    def __init__(self, trig, echo):
        self.trig = trig
        self.echo = echo

    def measure_cm(self, timeout_ms=30):
        """Single measurement; -1 on timeout."""
        self.trig.low()
        time.sleep_us(2)
        self.trig.high()
        time.sleep_us(10)
        self.trig.low()

        # Wait for Echo rising edge
        if not wait_high(self.echo, timeout_ms):
            return -1
        t_start = time.ticks_us()
        if not wait_low(self.echo, timeout_ms):
            return -1
        t_end = time.ticks_us()
        us = t_end - t_start
        return us / 58.0

    def measure_filtered(self, n=5):
        samples = []
        for _ in range(n):
            d = self.measure_cm()
            if d > 0:
                samples.append(d)
        if not samples:
            return -1
        return statistics.median(samples)


us = Ultrasonic(trig_pin, echo_pin)
while True:
    cm = us.measure_filtered()
    print("distance: %.1f cm" % cm)
    time.sleep_ms(100)
```


## 15. Debugging Methods

1. Test against a 1 m wall and verify the distance
2. Point at a distant wall and check stability
3. Observe the temperature effect on the sound speed
4. Check the Trig waveform (10 µs pulse)
5. Verify the Echo pulse width matches the distance


## 16. Frequently Asked Questions

- **HC-SR04 always returns 0**: unstable power or Trig / Echo swapped
- **5V Echo burned the MCU**: level shifting is mandatory
- **Poor distance accuracy**: no temperature compensation, target surface too oblique
- **Inaccurate liquid level**: container opening too small causes multiple reflections
- **Obstacle-avoidance distance jitter**: apply median filtering
- **HC-SR04 distance near zero**: blind zone; keep > 2 cm
