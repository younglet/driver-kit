# Button


## 1. Module Overview

A button (Button/Switch) is the most fundamental human-machine input component. Common use cases:

- Home appliance control panels (TV, air conditioner, washing machine);
- Industrial control panels;
- Instruments and meters;
- Remote controllers;
- Toys, game controllers;
- Keyboard, mouse;
- Embedded devices (Reset button).

Types:

- **Mechanical button** (tactile switch): momentary contact, conducts when pressed, breaks when released;
- **Latching switch**: stays pressed when pushed, releases when pushed again;
- **Slide switch**: toggle switch, state is retained;
- **Membrane switch**: copper foil button on PCB;
- **Capacitive touch button**: see capacitive touch document;
- **Rotary encoder**: see rotary encoder document.


## 2. Working Principle and Use Cases

Mechanical buttons conduct current through metal spring contacts:

- Conducted (pressed): contacts close;
- Open (released): contacts spring open;
- Bouncing: metal spring contacts produce multiple open/close transitions during contact (a few ms ~ tens of ms);
- Lifetime: 10⁵~10⁷ cycles (metal spring fatigue).

## 5. Hardware Connection

### 5.1 Basic Circuit

- **Pull-up resistor**: one end of the button connected to VDD (through resistor), other end to MCU;
- **Pull-down resistor**: one end of the button connected to GND (through resistor), other end to MCU;
- Internal pull-up: most MCUs have internal pull-up (30~50 kΩ), external resistor can be omitted;
- Pressed: GPIO low (active low) or high (active high);
- Series resistor: 100Ω current limiting to protect MCU.

### 5.2 Matrix Keypad

- Row lines: MCU output (drive low for scanning);
- Column lines: MCU input (pull-up);
- Diodes: prevent ghosting (multiple keys pressed simultaneously);
- 8 lines can scan 4x4 = 16 keys.

### 5.3 Debouncing

- Hardware RC: 10 nF capacitor + 1 kΩ resistor = 10 µs time constant;
- Hardware debounce IC: MAX6816 etc.;
- Software debounce: see below.

### 5.4 ESD Protection

- TVS diodes;
- 100Ω series resistor;
- Filter capacitors.


## 6. Communication Method

- Single GPIO: single button;
- Matrix: row-column scanning;
- I2C/SPI: touch IC;
- USB HID: keyboard (see USB document);
- Bluetooth HID: wireless keyboard;
- Infrared: remote control (NEC protocol etc.).


## 7. Initialization Flow

1. Configure GPIO (input + pull-up/pull-down);
2. Configure interrupt (rising/falling edge);
3. Configure timer (debounce, scanning);
4. Start.


## 8. Common Driver Interfaces

```python
class Button:
    """Single button."""
    def __init__(self, pin, active_low=True, debounce_ms=20):
        self.pin = pin
        self.active_low = active_low
        self.debounce_ms = debounce_ms
        self.pressed = False
        self.last_state = False
        self.last_time = 0

    def begin(self):
        pass

    def is_pressed(self):
        raw = self.pin.value()
        current = (not raw) if self.active_low else raw
        return current and self._stable(current)

    def _stable(self, state):
        now = time.ticks_ms()
        if state != self.last_state:
            self.last_state = state
            self.last_time = now
            return False
        return time.ticks_diff(now, self.last_time) > self.debounce_ms
```

- **Click**: press and release (within 200ms);
- **Double-click**: two consecutive clicks within a short time;
- **Long press**: press held longer than N seconds (1~3 seconds);
- **Extra-long press**: held longer than N seconds (e.g. 5 seconds);
- **Holding**: continuously held.

Implementation: state machine.



```python
class MultiButton:
    def __init__(self, pin, long_press_ms=1000):
        self.pin = pin
        self.long_press_ms = long_press_ms
        self.press_start = None

    def update(self):
        current = self.pin.value()
        if current and self.press_start is None:
            # Pressed
            self.press_start = time.ticks_ms()
        elif not current and self.press_start is not None:
            # Released
            duration = time.ticks_diff(time.ticks_ms(), self.press_start)
            self.press_start = None
            if duration > self.long_press_ms:
                return "LONG"
            return "CLICK"
        return None
```



```python
class MatrixKeypad:
    def __init__(self, row_pins, col_pins):
        self.row_pins = row_pins
        self.col_pins = col_pins

    def scan(self):
        keys = []
        for r in range(len(self.row_pins)):
            self.row_pins[r].low()  # Drive current row low
            for c in range(len(self.col_pins)):
                if self.col_pins[c].value() == 0:
                    keys.append((r, c))
            self.row_pins[r].high()
        return keys
```

## 10. Calibration, Compensation, and Filtering

```python
def debounce(pin, last_state, last_time, ms=20):
    current = pin.value()
    if current != last_state:
        if time.ticks_diff(time.ticks_ms(), last_time) > ms:
            return current, current, time.ticks_ms()
        return None, last_state, last_time
    return None, last_state, last_time
```


## 11. Interrupts, DMA, FIFO, and Buffers

- Single button: interrupt (press / release) + timer debounce;
- Matrix keypad: periodic timer scan (e.g. 10 ms);
- Long press: timer tracks press duration.


## 12. Error Handling

- **Bouncing**: hardware / software debounce;
- **Ghosting (matrix)**: multiple keys pressed simultaneously, diodes to prevent or software resolve;
- **Accidental trigger**: ESD interference, add decoupling and current limiting;
- **Stuck**: aged mechanical button, needs replacement;
- **Contact oxidation**: low-current load failure, use gold contacts or "wet circuit".


## 13. Low Power

- Button interrupt wakes up MCU (from deep sleep);
- Internal pull-up resistor is very power-efficient;
- Stop scanning timer when not in use.

## 15. Debugging Methods

1. Use oscilloscope to view button waveform (mechanical bounce);
2. Use logic analyzer to record press / release time;
3. Measure GPIO level change;
4. Measure contact resistance;
5. Test lifetime (button life tester).


## 16. Frequently Asked Questions

- **Accidental trigger**: bounce not eliminated, ESD interference;
- **No response**: GPIO configuration wrong, pull-up/pull-down not configured;
- **Double-click invalid**: debounce time set too long;
- **Long press too sensitive**: duration threshold not set properly;
- **Matrix ghosting**: missing diodes;
- **Mechanical stuck**: button aged or damaged.
