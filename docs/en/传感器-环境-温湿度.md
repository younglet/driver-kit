# Temperature & Humidity Sensor


## 1. Module Overview

A temperature & humidity sensor measures ambient temperature and relative humidity (RH%). It is commonly used in weather stations, air conditioning, agricultural greenhouses, smart homes, warehouse monitoring, and medical equipment.

Types: resistive, capacitive, and semiconductor. Capacitive is the most common (DHT series, SHT series, BME series).


## 2. Working Principle and Use Cases

- **Humidity-sensitive capacitor**: the sensing film absorbs water molecules, changing its dielectric constant and therefore the capacitance
- **Temperature sensor**: silicon bandgap temperature sensor or thermistor
- **Dew-point calculation**: compute the dew-point temperature from temperature and humidity to prevent condensation

Applications: weather monitoring, smart-home HVAC, greenhouse cultivation, museum / archive climate control, server rooms, pharmaceutical storage, personal weather stations.

## 5. Hardware Connection

- DHT11/22 single-bus: DATA pin needs a 4.7kΩ pull-up to VCC
- I2C pull-ups: 4.7kΩ
- Stay away from heat sources: the sensor does not self-heat but is heated by the PCB
- Prevent condensation: humidity sensors may be damaged by condensation
- Waterproof version: with PTFE membrane (SHT3x waterproof)
- I2C addresses: BME280 default 0x76 (SDO=high → 0x77); SHT3x default 0x44 (ADDR high → 0x45)


## 6. Communication Method

### 6.1 Single-bus (DHT11/22)

Host sends start signal (pull low 18 ms) → sensor responds (pull low 80 µs, pull high 80 µs) → 40-bit data (5 bytes: humidity int / decimal, temperature int / decimal, checksum).

### 6.2 I2C

Standard 100 / 400 kHz, simple timing.

### 6.3 SPI

Supported by a few chips (e.g. BME280).


## 7. Initialization Flow

1. Configure the I2C bus
2. Read WHO_AM_I (if available)
3. Soft-reset
4. Set measurement mode (single / continuous)
5. Set precision tier (high / medium / low)
6. Set measurement parameters (temperature, humidity, pressure)
7. Trigger measurement
8. Wait for data-ready
9. Read compensation values (e.g. SHT4x)

## 9. Data Format and Units

- Temperature: °C
- Humidity: %RH (0~100%)
- Dew point: °C, computed from temperature and humidity: `td = (b * γ(T, RH) + c) / (a - γ(T, RH))`
- Byte order: MSB first
- Update cycle: single measurement 10~15 ms; continuous per ODR


## 10. Calibration and Filtering

- Most chips are factory-calibrated
- High-precision needs: calibrate against a standard temperature/humidity source (e.g. thermostatic bath, salt solution)
- Save calibration values with version + CRC32
- Annual drift of 0.5~1% RH — periodic recalibration recommended


## 11. Error Handling

- Checksum error (common with DHT): strict timing not followed, retry
- I2C timeout: check address and pull-ups
- Data saturation: temperature/humidity out of range
- Dew condensation: sensor immersed in water must be dried or replaced
- Long-term drift: recalibrate periodically


## 12. Low Power

- Automatically enters sleep after single measurement
- Measurement current ~0.5 mA, sleep < 1 µA
- Suitable for battery-powered intermittent wake-up (1 sample / second)

## 14. Debugging Methods

1. Check chip pin voltages
2. Read WHO_AM_I to verify communication
3. Compare at room temperature: hold the sensor and observe the temperature rise
4. Breathe on the sensor and observe humidity increase
5. Ice water for 0°C, hot water for 100°C cross-check
6. Compare dew point with weather apps


## 15. Frequently Asked Questions

- **DHT22 always fails**: single-bus timing has strict µs-level requirements; use a timer or DWT delay
- **DHT11 returns all zeros**: pull-up resistor missing
- **Inaccurate humidity**: long storage in a humid environment; recovery may take several hours
- **High temperature reading**: chip heated by the PCB — move to a ventilated spot
- **I2C error**: wrong address (DHT series do not use I2C)

## 17. Dew Point and Absolute Humidity

```python
import math

def dew_point(t, rh):
    """Magnus formula approximation"""
    a, b, c = 17.27, 237.7, 17.5
    gamma = (a * t / (b + t)) + math.log(max(rh / 100.0, 1e-6))
    return b * gamma / (a - gamma)
```

Absolute humidity (g/m³) = 216.7 × (RH/100) × 6.112 × exp(17.62 × T / (243.12 + T)) / (273.15 + T)
