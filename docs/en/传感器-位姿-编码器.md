# Encoder


## 1. Module Overview

An encoder is a position / angle sensor that converts mechanical displacement (rotary or linear) into electrical signals. It is widely used in motor servos, robot joints, conveyor lines, CNC machines, and 3D printers.

Main types:
- **Incremental encoder**: outputs A/B phase pulses and counts while rotating
- **Absolute encoder**: each position maps to a unique code, retained without power
- **Linear encoder**: measures linear displacement
- **Optical encoder**: grating + optoelectronic element
- **Magnetic encoder**: Hall / AMR / TMR sensing a magnetic ring
- **Capacitive encoder**: detects capacitance variation


## 2. Working Principle and Use Cases

### 2.1 Incremental

Internal grating / magnetic ring with evenly spaced markings. Rotation produces two quadrature square waves A/B with a 90° phase difference. Rotation direction is determined by whether A leads or lags B. Each revolution yields N pulses (PPR / CPR).

Common applications:
- DC motor speed measurement
- Closed-loop stepper motor
- Manual knobs (volume)
- CNC spindle
- Conveyor belt distance measurement

### 2.2 Absolute

Each mechanical position corresponds to a unique code value (Gray code or binary). Interfaces include SPI, SSI, BISS, RS485, CAN.

Applications:
- Robot joints (high precision, position retained through power loss)
- CNC machines
- Industrial automation

### 2.3 Magnetic Encoder

A magnet is mounted on the rotating shaft; a magnetic-sensitive element (Hall / AMR / TMR) detects the field direction. Compact and contamination-resistant.

Applications: in-motor (BLDC, PMSM), robot joints, drone gimbals.

| Operating temperature | -40°C ~ +85°C |

**Selection suggestions**:
- Knobs / small projects: EC11 / EC12 (mechanical) / AS5600 (magnetic)
- Motor servo: incremental AB phase + index Z
- Robot joints: high-precision absolute (AS5048A, AEAT, RLS)
- Industrial CNC: optical encoder + linear scale


## 5. Hardware Connection

- AB phase differential output: use an RS422 receiver (e.g. AM26LS32, MAX3094)
- Power decoupling: 100nF + 10µF
- Use shielded twisted-pair cable to reduce EMI
- Magnetic encoders: align the magnet to the chip (radial vs axial magnetization)
- Long-distance: use shielded cable
- Grounding: single-point ground to avoid loops


## 6. Communication Method

- Incremental AB phase: square-wave signals captured by hardware timers
- Incremental + index Z: one zero-reference pulse per revolution
- SSI (Synchronous Serial): CLK + DO, one-way absolute
- BISS: SSI upgrade with bidirectional communication and parameter read-back
- SPI: supported by some absolute chips
- I2C: AS5600, AS5601, etc.
- Analog / PWM: low-cost option


## 7. Initialization Flow

1. Configure I2C (default address 0x36)
2. Verify WHO_AM_I (if present)
3. Set the zero position (write the ZPOS register)
4. Set the maximum angle (MPOS)
5. Set the output mode (ANG / RAW)
6. Configure PWM frequency (if using PWM output)
7. Start reading the angle

### 10.1 M-method (frequency measurement)

Count pulses within a fixed time window. Poor accuracy at low speeds.

```python
pulses = encoder.read_pulses()
rpm = pulses * 60 / (cpr * dt)
```

### 10.2 T-method (period measurement)

Measure the time between adjacent pulses. Poor accuracy at high speeds.

### 10.3 M/T method

Combines both; good accuracy across a wide speed range.


## 9. Data Format and Units

- Incremental: pulse count / CPR
- Absolute: bit count (10/12/14/16/18/20/24 bit)
- Angle units: degrees (°), radians (rad), counts
- Angular velocity units: rpm (revolutions per minute), dps (degrees per second)
- Data range: 0~360° / -180°~+180° / 0~2π


## 10. Calibration and Filtering

- Median filter: remove pulse glitches
- Moving average: smooth velocity jitter
- Kalman filter: state estimation (angle + velocity)


## 11. Interrupts and DMA

- Incremental: hardware-timer capture interrupt
- High-speed applications: DMA + timer
- Absolute: on-demand read


## 12. Error Handling

- Encoder cable break: check AB phase levels
- Zero lost: battery drain causes the absolute position to be lost
- Count overflow: use 32-bit counters
- Signal interference: shielded cable + filter capacitors


## 13. Low Power

- The encoder itself usually has no low-power mode
- The host can disable timer interrupts to reduce wake-ups


## 14. Example Code

```python
class Encoder:
    def __init__(self, cpr=360):
        self.cpr = cpr
        self.count = 0

    def begin(self):
        pass

    def update(self, delta):
        """Called from interrupt: accumulate count."""
        self.count += delta

    def read_angle(self):
        """Return 0~360°."""
        return (self.count % self.cpr) * 360.0 / self.cpr

    def read_velocity(self, dt):
        """Return rpm."""
        return self.count / self.cpr * 60 / dt

    def reset(self):
        self.count = 0


enc = Encoder(cpr=360)
enc.begin()
while True:
    deg = enc.read_angle()
    print("angle=%.1f°" % deg)
```


## 15. Debugging Methods

1. Rotate by hand and watch the count change
2. Slow steady rotation — verify smooth velocity
3. Fast forward/reverse rotation — verify direction
4. Check the AB phase signal waveform with a scope
5. Verify the index Z zero-reference signal


## 16. Frequently Asked Questions

- **Wrong direction**: swap A/B wiring or use software reverse
- **Count jumps**: mechanical vibration / chatter; add hardware filtering or software debounce
- **Zero lost**: battery low on the absolute encoder
- **Insufficient accuracy**: low line count, sampling jitter, magnet position offset
