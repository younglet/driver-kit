# 3-Axis Gyroscope


## 1. Module Overview

A 3-axis gyroscope measures angular velocity around the X, Y, and Z axes. It is widely used in robotics, drones, gimbals, game controllers, wearables, and motion detection.

The output is angular velocity, not angle. The angle can be estimated by integration, but this accumulates drift over time, so long-term attitude estimation usually requires sensor fusion with an accelerometer, magnetometer, GNSS, or visual odometry.


## 2. Working Principle and Use Cases

```text
GX: angular velocity around the X axis
GY: angular velocity around the Y axis
GZ: angular velocity around the Z axis
```

Most chips adopt a right-handed coordinate system, but the actual axis mapping, sign convention, and mounting direction must be verified against the chip datasheet.

Common units:

- `°/s`: degrees per second
- `rad/s`: radians per second
- Raw register counts

## 4. Key Parameters and Selection Guide

### 4.1 Key Parameters

| Parameter | What to Check |
|---|---|
| Range | ±125, ±250, ±500, ±1000, ±2000 °/s, etc. |
| Noise | Short-term output stability |
| Bias stability | Long-term drift at rest |
| Output data rate | Whether it meets the control-loop frequency |
| Interface | I2C is convenient; SPI typically offers higher speed and real-time performance |
| FIFO | Reduces MCU read jitter and sample loss |
| Interrupts | Data-ready, motion, and wake-up events |
| Power | Supply voltage, IO level, peak current |
| Temperature range | Environmental adaptability and temperature drift |
| Supply chain | EOL status, reliable sourcing |

### 4.2 Selection Suggestions

- **Beginner and general robotics**: MPU-6050, BMI160, LSM6DS3
- **Low-power devices**: BMI270, LSM6DSL, LSM6DSO, ICM-42605
- **High-speed motion and attitude control**: ICM-20602, ICM-42605, ICM-42688-P
- **Absolute heading required**: choose a 9-axis IMU or add an external magnetometer, but watch for magnetic interference from motors and ferromagnetic materials
- **Mass production**: prioritize original-part status, lifecycle, package, layout constraints, and long-term supply rather than module price alone.


## 5. Hardware Connection

- Confirm the chip supply voltage and IO logic level
- Do not connect 5V signals directly to 1.8V/3.3V-only IO
- I2C requires proper pull-up resistors
- When sharing SPI among multiple devices, give each device its own chip select
- Keep the sensor away from high-current switching nodes and strong vibration sources
- Place power decoupling capacitors close to the supply pins
- Mechanical mounting must be firm to avoid self-induced vibration
- Production designs should follow the manufacturer's recommended PCB layout

Modules may not all be 5V-tolerant. On-board regulators, level shifters, and I2C address configurations can also vary between module revisions — check the schematic or measure before use.


## 6. Initialization Flow

A typical construction pattern:

```python
imu = Gyroscope(i2c, address=0x68)
```

Common initialization and diagnostic interfaces:

```python
imu.begin()
imu.reset()
imu.is_connected()
imu.is_ready()
imu.who_am_i()
imu.self_test()
```

Initialization usually performs the following:

1. Configure I2C or SPI
2. Check the device ID
3. Reset and exit sleep mode
4. Set range, sample rate, and filter
5. Clear FIFO and interrupt status
6. Run any necessary self-tests

Read interfaces:

```python
gx, gy, gz = imu.read_gyro()
raw_x, raw_y, raw_z = imu.read_raw_gyro()
```

Configuration interfaces:

```python
imu.set_gyro_range(500)
imu.set_sample_rate(100)
imu.set_filter_bandwidth(42)
```

Status and power interfaces:

```python
imu.data_ready()
imu.sleep()
imu.wake()
imu.read_temperature()
```

Function names vary across libraries; refer to the actual driver implementation for the exact API.


## 7. Data Format and Units

Raw data is usually a signed integer; the typical conversion is:

```text
angular velocity = raw value / sensitivity
```

The sensitivity MUST come from the chip datasheet. The driver must clearly state:

- Unit
- Signed vs unsigned
- Endianness (byte order)
- Scaling factor
- Data update time
- Timeout conditions

Polling example:

```python
if imu.data_ready():
    gx, gy, gz = imu.read_gyro()
```

When using FIFO, check the count and overflow flag to ensure each frame has a consistent length. High sample-rate applications can use a data-ready interrupt or DMA and compute the sample period with real timestamps.


## 8. Calibration and Filtering

During calibration:

1. Keep the device completely still
2. Collect multiple samples
3. Compute the mean bias on each axis
4. Add a version and checksum when saving the calibration values
5. Recalibrate or apply temperature compensation when the temperature changes significantly

```python
bias_x, bias_y, bias_z = imu.calibrate_gyro(samples=500)
```

Axis confirmation method: rotate around each of the X, Y, Z axes and observe which channel changes the most; use the right-hand rule to confirm the sign. Do not assume different modules share the same axis orientation.

Small ranges typically have higher sensitivity but saturate easily; large ranges suit fast rotation. A low-pass filter reduces noise but introduces latency.


## 9. Error Handling

At minimum, handle the following:

- Device not responding
- `WHO_AM_I` mismatch
- I2C bus stuck
- SPI chip-select timing errors
- Communication timeout
- Data not ready
- FIFO overflow
- Raw value out of range
- Device power-loss or reset
- Device moved during calibration

Recommended flow: set a communication timeout, retry a bounded number of times; on continued failure perform bus recovery, sensor reset, and re-initialization; after repeated failures report the fault to the application layer rather than blocking forever.

When receiving or inspecting parts:

1. Read the device ID
2. Check the static bias
3. Rotate around each of the three axes
4. Check the I2C address
5. Spot-check batches, especially for low-cost or suspect modules


## 10. Low Power

```python
imu.sleep()
# Wake via GPIO or interrupt
imu.wake()
```

Before sleeping, confirm whether biases must be saved, whether FIFO and interrupt status will be cleared, and whether registers need to be reconfigured after wake-up. Low-power selection should compare sleep current, wake-up latency, and low-power data rate — not just the operating current.

```python
        self.bus = bus
        self.address = address
        self.bias = (0.0, 0.0, 0.0)

    def begin(self):
        pass

    def read_gyro(self):
        """Return gx, gy, gz in °/s."""
        pass

    def read_raw_gyro(self):
        pass

    def set_gyro_range(self, dps):
        pass

    def set_sample_rate(self, hz):
        pass

    def calibrate_gyro(self, samples=500):
        pass

    def get_corrected_gyro(self):
        gx, gy, gz = self.read_gyro()
        bx, by, bz = self.bias
        return gx - bx, gy - by, gz - bz

imu = Gyroscope(i2c, address=0x68)
if not imu.begin():
    raise RuntimeError("Gyroscope initialization failed")

imu.set_gyro_range(500)
imu.set_sample_rate(100)
imu.calibrate_gyro(samples=500)

while True:
    if imu.data_ready():
        gx, gy, gz = imu.get_corrected_gyro()
        print("gx=%.2f, gy=%.2f, gz=%.2f °/s" % (gx, gy, gz))
```
