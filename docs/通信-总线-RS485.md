# RS485


## 1. 模块概览

RS485 是 EIA/TIA-485 标准定义的差分串行通信接口，用于工业环境下多点通信。常见于：
- 工业自动化（PLC、变频器、伺服）；
- 仪表（电表、水表、气表）；
- 楼宇自控（BA）；
- 传感器网络；
- 长距离通信（< 1200 m）；
- 抗噪声环境。

特点：
- 差分信号（A/B 线）；
- 半双工或全双工；
- 总线拓扑（多设备）；
- 长距离、高速率；
- 抗共模干扰；
- 工业级抗扰度。


## 2. 工作原理与适用场景

### 2.1 物理层

- **A 线**：反相（inverting），差分负端；
- **B 线**：同相（non-inverting），差分正端；
- **差分电压**：|VA - VB| > 200 mV = 逻辑 1；< -200 mV = 逻辑 0；
- **共模电压**：-7V~+12V（驱动器能承受）；
- **总线电压**：典型 0~5V。

### 2.2 拓扑

- 总线型（菊花链），不能星型；
- 终端电阻 120Ω（总线两端）；
- 收发器控制（DE/RE）：半双工需要切换方向。

### 2.3 速率 vs 距离

| 速率 | 距离 |
|---|---|
| 10 Mbps | 15 m |
| 1 Mbps | 120 m |
| 100 kbps | 1200 m |
| 10 kbps | 12000 m |

## 5. 硬件连接

### 5.1 接线

- A、B 双绞线（推荐）；
- 总线两端接 120Ω 终端电阻；
- 总线不能星型拓扑（信号反射）；
- 不能短距离多分支；
- 共地（一般建议单点接地）。

### 5.2 终端电阻

- 总线两端各一个 120Ω 电阻（与线缆特性阻抗匹配）；
- 多分支或短线缆可省（阻抗不匹配影响小）；
- 120Ω + 1 nF（抗高频反射）。

### 5.3 收发控制（DE/RE）

```text
发送：DE=高，RE=高，驱动器使能
接收：DE=低，RE=低，接收器使能
```

- 半双工：DE 和 RE 一起控制（多数芯片）；
- 全双工：无 DE/RE，发送和接收同时工作；
- 自动方向芯片：硬件自动切换，无需 MCU 控制。

### 5.4 隔离

- 地环路干扰：长距离、多设备场景常见；
- 隔离方案：DC-DC 隔离电源 + 光耦/数字隔离芯片（ISO3082、ADuM1411）；
- 推荐：工业应用必加隔离；
- 注意：隔离两侧供电独立（电源隔离）。

### 5.5 保护

- TVS 二极管（防浪涌）；
- PTC 自恢复保险丝；
- 共模扼流圈（CMCC）；
- ESD 保护：±15 kV 静电（部分工业收发器内置）。


## 6. 通信方式

- 半双工（A、B 共享）；
- 全双工（A、B、Y、Z 双对）；
- 主从协议（Modbus 常用）；
- 多主机协议（自定义）；
- 多点轮询。


## 7. 初始化流程

1. 配置 GPIO（TX、RX、DE/RE）；
2. 配置 UART（波特率、数据位、停止位、校验）；
3. 配置 DE/RE 默认为接收；
4. 测试（发送 0x55 看波形）；
5. 开始通信。


## 8. 驱动接口

```python
class RS485:
    def __init__(self, uart, de_pin):
        self.uart = uart
        self.de = de_pin

    def begin(self):
        self.de.low()  # 默认接收

    def write(self, data):
        """发送数据。"""
        self.de.high()  # 发送模式
        self.uart.write(data)
        self.uart.flush()
        self.de.low()  # 切换到接收

    def read(self, n=1):
        """读数据。"""
        return self.uart.read(n)

    def any(self):
        return self.uart.any()


class ModbusRTU:
    """Modbus RTU 主机。"""
    def __init__(self, rs485):
        self.rs485 = rs485
        self.timeout = 0.1

    def read_holding(self, slave, addr, qty):
        """读保持寄存器（功能码 0x03）。"""
        frame = self._build_frame(slave, 0x03, addr, qty)
        self.rs485.write(frame)
        return self._parse_response()

    def write_register(self, slave, addr, value):
        """写单个寄存器（功能码 0x06）。"""
        frame = self._build_frame(slave, 0x06, addr, value)
        self.rs485.write(frame)

    def _crc16(self, data):
        """Modbus CRC16。"""
        crc = 0xFFFF
        for byte in data:
            crc ^= byte
            for _ in range(8):
                if crc & 1:
                    crc = (crc >> 1) ^ 0xA001
                else:
                    crc >>= 1
        return crc
```


## 9. 数据格式与单位

- 字节流（UART 帧格式）；
- 协议层：Modbus RTU（详见 Modbus 文档）；
- 帧间隔：3.5 字符时间（Modbus RTU）；
- 地址：从机号（1~247）；
- 功能码 + 数据 + CRC16。


## 10. 中断与 DMA

- 接收：中断 + 环形缓冲；
- DMA：连续接收（大数据）；
- 发送：FIFO + 完成后切换到接收；
- 总线空闲检测：3.5 字符时间无数据 = 帧结束。


## 11. 错误处理

- **校验错误（CRC）**：重新发送；
- **无响应**：从机地址错、速率错、断线；
- **总线冲突**：多主机同时发送，仲裁；
- **超时**：主机等待响应时间过长；
- **噪声干扰**：增加屏蔽、使用隔离；
- **终端电阻缺失**：信号反射造成误码。


## 12. 低功耗

- 不通信时切换到接收模式；
- 收发器待机电流 < 1mA；
- 整机可关闭 RS485 模块。


## 13. 示例代码

```python
rs485 = RS485(uart, de_pin)
rs485.begin()

# 发送
rs485.write(b'\x01\x03\x00\x00\x00\x06\xC5\xC8')

# 接收
if rs485.any():
    data = rs485.read()
    print(data)
```


## 14. 调试方法

1. 测 A、B 波形（示波器差分探头）；
2. 检查终端电阻（120Ω）；
3. 检查 DE/RE 切换时序；
4. 单独测试收发（自环）；
5. 用 USB-RS485 转换器连接 PC 调试；
6. 检查隔离电源（隔离版本）。


## 15. 常见问题

- **收不到数据**：DE/RE 反、A/B 反、终端电阻错；
- **数据错误**：终端电阻错、距离过长、速率过高；
- **部分从机无响应**：地址错、波特率错；
- **多设备冲突**：设备地址重复；
- **总线不稳定**：隔离未加、屏蔽未做；
- **长距离通信失败**：速率过高、加中继器。


## 16. 参考资料

- 总线两端必须有终端电阻（多分支结构复杂时需仔细设计）；
- 隔离电源必须独立；
- 远离强干扰源（变频器、大功率电机）；
- 总线速率与距离成反比。
