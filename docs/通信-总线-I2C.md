# I2C（IIC）


## 1. 模块概览

I2C（Inter-Integrated Circuit，集成电路总线）是 Philips（现 NXP）于 1982 年开发的同步串行通信总线，用于芯片间短距离通信。

特点：
- 仅 2 根信号线（SDA、SCL）；
- 多主多从架构；
- 7-bit 或 10-bit 地址；
- 速度档位：标准 100 kHz、快速 400 kHz、快速+ 1 MHz、高速 3.4 MHz；
- 应用：传感器、EEPROM、RTC、I/O 扩展、显示屏、音频编解码器等。


## 2. 工作原理与适用场景

### 2.1 物理层

- **SDA**：数据线（双向）；
- **SCL**：时钟线（主机驱动）；
- **开漏 + 上拉**：所有设备只能拉低，总线高电平靠上拉电阻；
- **上拉电阻**：4.7 kΩ（100 kHz 标准）/ 2.2 kΩ（400 kHz）/ 1 kΩ（1 MHz+）；
- **最大电容**：400 pF（限制设备数量和线长）。

### 2.2 协议层

- **起始位（START）**：SDA 在 SCL 高电平时由高拉低；
- **停止位（STOP）**：SDA 在 SCL 高电平时由低拉高；
- **数据位**：SCL 高电平时 SDA 状态有效；
- **ACK/NACK**：第 9 个 SCL 周期，从机拉低 SDA = ACK，否则 NACK；
- **重复起始（Repeated START）**：不发 STOP 直接发 START；
- **字节格式**：MSB 在前，每字节 8 bit + 1 ACK。

### 2.3 帧格式

```text
[S] [地址+R/W] [ACK] [数据1] [ACK] ... [数据N] [ACK/NACK] [P]
```

- 7-bit 地址：1 byte 地址（7 位地址 + 1 位 R/W）；
- 10-bit 地址：前缀 11110xx + 8 位地址 = 2 字节地址。

| 参数 | 说明 |

**选型建议**：
- 100 kHz：通用传感器、低速 EEPROM；
- 400 kHz：常用速度，多数传感器支持；
- 1 MHz：快速传感器、OLED 显示；
- 3.4 MHz：极少使用，主机需特殊支持。


## 5. 硬件连接

### 5.1 接线

- SDA + SCL 两根信号线 + GND 共地；
- 上拉电阻：SDA 和 SCL 各一上拉到 VCC；
- 电源 + 地：每个设备单独供电（部分共享）；
- 板级 I2C：上拉到 MCU VDD（3.3V）；
- 多电压：电平转换芯片（如 PCA9306）。

### 5.2 上拉电阻选择

```text
R_min = (Vcc - V_OL_max) / I_OL_min
     = (3.3V - 0.4V) / 3mA ≈ 1 kΩ
R_max = t_r / (0.8473 × C_b)
t_r = 上升时间（ns，标准 1000 ns）
C_b = 总线电容（pF）
例：C_b = 100 pF → R_max ≈ 11.8 kΩ
```

所以常用 **4.7 kΩ** 是大多数场景的折衷。

### 5.3 走线

- 短而直（< 30 cm 最佳）；
- 远离噪声源（DC-DC、电机）；
- 屏蔽线（长距离或强干扰环境）；
- 不能与电力线平行走线。

### 5.4 上拉电源选择

- 板级上拉：3.3V（与 MCU 一致）；
- 多电压系统：每个电压域独立上拉 + 电平转换；
- 总线监控：加 ESD 保护（如 USBLC6）。


## 6. 通信方式

- 主机发起（SCL 由主机驱动）；
- 多主机仲裁（线与逻辑）；
- 时钟同步（多主机）；
- 时钟拉伸（slave hold SCL low 等待就绪）。


## 7. 初始化流程

1. 配置 GPIO（SDA/SCL 开漏）；
2. 启用上拉（内部或外部）；
3. 配置 I2C 控制器（地址、速度）；
4. 测试总线（扫描地址）；
5. 开始通信。

## 9. 数据格式与单位

- 数据：字节流（MSB 在前）；
- 帧：`[START] [ADDR+R/W] [ACK] [DATA] [ACK] [DATA] [ACK/NACK] [STOP]`；
- 寄存器读写：先写寄存器地址（MSB 优先或 LSB 优先，设备决定）；
- 多字节：高位在前（多数）。


## 10. 校准与滤波

- 时钟拉伸：从机拉低 SCL 等待就绪；
- 主机需支持时钟拉伸；
- 总线超时保护：避免从机无限拉伸。


## 11. 中断与 DMA

- 多数 I2C 设备不支持硬件中断（需用 GPIO 中断）；
- 部分 MCU 内部 I2C FIFO 支持连续传输。


## 12. 错误处理

- **NACK**：从机无应答，检查地址、上拉；
- **总线卡死**：SDA/SCL 被从机拉低，发送 9 个 SCL 脉冲复位；
- **仲裁失败**：多主机冲突；
- **超时**：总线超时检测；
- **CRC8（I2C V1.1）**：新增 CRC 校验（部分设备支持）；
- **SMBus Alert**：从机主动通知主机。

### 12.1 总线恢复

```python
def recover_bus(sda, scl):
    """发送 9 个 SCL 脉冲复位从机。"""
    for _ in range(9):
        scl.high()
        scl.low()
    sda.high()
```


## 13. 低功耗

- 关闭未使用的 I2C 外设；
- 待机电流 < 1µA（多数 I2C 设备）；
- 主机休眠时停止总线；
- 设备支持硬件地址唤醒。


## 14. 示例代码

```python
i2c = I2C(sda_pin, scl_pin, freq=400000)
i2c.begin()
print("扫描总线:", i2c.scan())

# 写寄存器
i2c.write_then_read(0x68, b'\x6B', 1)  # 读 MPU6050 PWR_MGMT_1
# 写寄存器
i2c.write_then_read(0x68, b'\x6B', b'\x00')  # 唤醒
```


## 15. 调试方法

1. 测 SDA/SCL 电压（上拉是否正常）；
2. 测 SCL 波形（示波器）；
3. 扫描总线地址（i2cdetect）；
4. 用逻辑分析仪抓波形；
5. 检查 ACK 是否正常；
6. 检查时钟拉伸（SCL 被拉低）。


## 16. 常见问题

- **无应答**：地址错、上拉未接、设备未上电；
- **卡死**：从机一直拉低 SDA，复位总线；
- **数据错误**：速度过快、线太长、EMI；
- **多设备地址冲突**：更换 ADDR 引脚；
- **I2C 时序错**：速度过快时序不准（高速模式）。
