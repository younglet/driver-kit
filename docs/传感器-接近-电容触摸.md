# 电容触摸传感器


## 1. 模块概览

电容触摸传感器通过检测人体或物体引起的电容变化实现触摸检测。类型：
- **自电容触摸**：单电极，检测电极与地之间的电容变化（人体感应）；
- **互电容触摸**：双电极（Tx/Rx），用于矩阵触摸和多点触控；
- **电阻触摸**：见显示部分；
- **专用触摸 IC**：内置检测算法；
- **MCU 内部触摸**：部分 MCU（如 ESP32）内置触摸 IO。

应用：
- 触摸按键（替代机械按键）；
- 触摸滑条；
- 矩阵键盘；
- 屏幕触控（手机/平板）；
- 防水面板（厨房电器）；
- 智能家居（玻璃面板）；
- 触控 IC（TTP229、MPR121、FT6336）。


## 2. 工作原理与适用场景

### 2.1 自电容

触摸电极与地构成电容 Cp。人体接近时增加电容 C_touch → 总电容变化：

```text
ΔC = C_touch - Cp
```

测量电路通过 RC 振荡、恒流源积分或频移检测电容变化。

### 2.2 互电容

Tx 与 Rx 电极交叉点形成耦合电容。触摸改变耦合 → 接收端信号变化。

### 2.3 MCU 内部触摸

如 ESP32 通过 IO 上拉电阻测量触摸电流，时间差异反映电容变化。

## 5. 硬件连接

- 触摸电极：PCB 上的铜箔或 ITO 玻璃/薄膜；
- 电极尺寸：5~30 mm 直径（或长方形）；
- 触摸走线：尽量短，远离噪声源；
- 电极覆盖层：玻璃、亚克力（0~10 mm 厚度）允许覆盖；
- 寄生电容：电极间需间距 1~5 mm 防止干扰；
- 接地：系统良好接地可降低误触发；
- 电源：稳压 + 去耦电容（100nF + 10µF）；
- 防水：使用矩阵扫描 + 防水算法（雨滴 vs 手指识别）。


## 6. 通信方式

- 数字高低：1 路触摸直接输出；
- I2C：MPR121、FT6336 等多键芯片；
- SPI：部分芯片支持；
- 中断输出：触摸事件触发 INT。


## 7. 驱动接口

```python
class TouchSensor:
    def begin(self): ...
    def reset(self): ...

    def set_threshold(self, touch, release): ...
    def calibrate(self): ...
    def sleep(self): ...
    def wake(self): ...

class TouchSlider:
    def begin(self): ...
    def read_position(self): ...  # 返回 0~100%
```



- 3 个电极滑条：两端为主、中间参考，输出位置百分比；
- 多电极滑条：12 个以上电极组成线性/圆形滑条；
- 灵敏度：电极越窄越敏感、但抗噪声差。


## 8. 数据格式与单位

- 数字输出：布尔（触摸/未触摸）；
- I2C 输出：触摸位图（如 MPR121 返回 12-bit 触摸状态）；
- 滑条/滚轮：百分比位置（0~100%）；
- 矩阵键盘：行/列号；
- 触控屏：X/Y 坐标 + 多点 ID。


## 9. 校准与滤波

- 触摸 IC 自动校准（环境温度、湿度漂移）；
- 启动后空闲状态下读基线；
- 阈值：触摸阈值（按下）和释放阈值（抬起）；
- 防水算法：阈值自适应，区分雨滴/手指；
- 滑条线性化：原始值 vs 物理位置曲线拟合。


## 10. 中断与 DMA

- 触摸中断：触摸状态变化触发；
- 多点触控：FT6336、GT911 支持最多 5~10 点；
- FIFO：部分芯片支持触摸序列记录。


## 11. 错误处理

- I2C 错误：检查地址、上拉；
- 误触发：阈值过低或电极受潮；
- 不响应：阈值过高或电极未连接；
- 漂移：温度/湿度变化大，自动重校准可缓解。


## 12. 低功耗

- 单键触摸 ~ 1 µA 静态电流；
- 多键芯片：触摸时 mA 级，待机 µA 级；
- 间歇扫描：1 Hz 足够应用。


## 13. 示例代码

```python
class MPR121:
    def __init__(self, bus, address=0x5A):
        self.bus = bus
        self.address = address

    def begin(self):
        pass

    def read_touched(self):
        """返回 12-bit 触摸位图。"""
        pass

    def is_touched(self, channel):
        """检测单通道触摸。"""
        return (self.read_touched() & (1 << channel)) != 0


mpr = MPR121(i2c, address=0x5A)
mpr.begin()
while True:
    state = mpr.read_touched()
    for i in range(12):
        if state & (1 << i):
            print("按钮 %d 被按下" % i)
```


## 14. 调试方法

1. 触摸电极观察 INT 触发；
2. 读取触摸位图确认状态；
3. 调节阈值观察灵敏度；
4. 抗干扰测试（手机干扰、电吹风）；
5. 防水测试（滴水）；
6. 接近测试（5~10 cm 距离）。


## 15. 常见问题

- **触摸不响应**：电极未连接、阈值过高、电极受潮；
- **误触发**：阈值过低、ESD 干扰、电极污渍；
- **滑条位置不准**：电极形状不均匀、阈值不当；
- **雨滴误触发**：防水算法未启用；
- **覆盖玻璃太厚**：> 10 mm 触摸灵敏度大幅下降，需使用专用 IC。

## 17. 初始化流程（MPR121 为例）

1. 配置 I2C（默认 0x5A，ADDR 接 VSS 0x5B）；
2. 软复位；
3. 配置电极充放电电流和时间；
4. 配置触摸阈值、释放阈值；
5. 配置去抖时间；
6. 配置自动重校准；
7. 启动触摸扫描；
8. 中断读取触摸状态。


## 18. 抗干扰和防水

- 抗电源干扰：滤波电容 + 软件滤波；
- 抗 ESD：TVS 二极管 + 串联电阻；
- 抗 RFI：屏蔽 + 软件滤波；
- 防水：阈值自适应，区分雨滴 vs 手指接触；
- 接近感应：检测距离 1~10 cm 的物体接近。
