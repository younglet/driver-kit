# DriverKit · 驱动派

> 嵌入式硬件驱动实战文档库 —— 53 个常用模块的原理、选型、接线、驱动和调试速查手册。

基于 [VitePress](https://vitepress.dev/) 构建，部署后访问示例站点即可阅读全部内容。

## 🚀 部署

站点通过 GitHub Pages + Actions 自动部署：

1. 推送代码到 `main` → 触发 `.github/workflows/deploy.yml`
2. Job 跑 `npm ci` + `npm run docs:build`，产物上传为 artifact
3. 第二个 Job 把 artifact 部署到 GitHub Pages
4. 首次启用需要在仓库 **Settings → Pages → Source** 选 `GitHub Actions`

部署后访问：**<https://younglet.github.io/driver-kit/>**

## 🚀 本地预览

需要 Node.js 18+。

```bash
npm install          # 安装依赖（VitePress）
npm run docs:dev     # 启动开发服务器（默认 http://localhost:5173）
npm run docs:build   # 构建静态站点到 docs/.vitepress/dist
npm run docs:preview # 本地预览构建产物
```

## ✍️ 新增 / 修改模块文档

> 本节面向**项目贡献者**，读者请忽略。

### 文件命名

```text
分类-领域-模块.md
```

示例：

- `传感器-位姿-三轴陀螺仪.md`
- `通信-总线-I2C.md`
- `执行器-运动-直流电机.md`

### 完整章节规范

每篇模块文档应遵循《总表-硬件驱动文档分类.md》中定义的 19 节结构（不适用的章节可省略），包括但不限于：

1. 模块概览
2. 工作原理和适用场景
3. 常见型号和参考价格
4. 关键参数和选型建议
5. 硬件连接和电气注意事项
6. 通信方式或控制信号
7. 初始化流程
8. 常见驱动接口
9. 数据格式、单位和时序
10. 配置参数
11. 校准、补偿和滤波
12. 中断、DMA、FIFO 或缓存
13. 错误处理和异常恢复
14. 低功耗
15. 示例代码
16. 调试方法
17. 常见问题
18. 开发注意事项
19. 参考资料

详细规范请阅读 [`总表-硬件驱动文档分类.md`](./总表-硬件驱动文档分类.md)。

### 添加新模块的步骤

1. **复制模板**：从结构最接近的现有模块复制一份 `.md` 文件作为模板
2. **重命名**：按命名规范 `分类-领域-模块.md` 重命名
3. **撰写内容**：按规范补全章节（型号表 ≥ 5 个、价格区间、合理代码示例）
4. **更新索引**：
   - 在 [`docs/分类总览.md`](./docs/分类总览.md) 中加入链接
   - 在 [`docs/.vitepress/config.mts`](./docs/.vitepress/config.mts) 的 `sidebar` 中加入该项
   - 如新增分类，在 `nav` 中加入对应大类入口

### 价格与参数规范

- **价格**：必须用人民币估算范围（如 `¥10~30`），并明确"仅用于预算参考"
- **参数**：重要规格（量程、精度、通信速率等）必须以原厂数据手册为准
- **示例代码**：可执行的伪代码或 Python 风格示例，标注单位
- **错误处理**：每章应有"常见问题"或"错误处理"小节，避免纯理论堆砌

### Pull Request 检查清单

- [ ] 文件命名符合 `分类-领域-模块.md`
- [ ] 章节结构与现有模块一致
- [ ] 包含至少 5 个常见型号 + 参考价格
- [ ] 代码示例可读（注明平台/库）
- [ ] `分类总览.md` 已更新链接
- [ ] `config.mts` 的 `sidebar` 已更新
- [ ] 本地 `npm run docs:build` 通过无报错
